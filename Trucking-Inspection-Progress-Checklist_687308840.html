VARLab : Trucking Inspection Progress Checklist

Created by Nguyen Ha Dao (Unlicensed), last modified on Apr 25, 2023

In Advanced Inspection, students need a way to keep track of what they’ve done so far and what they have left to do. The inspection is long enough that students might often exit partway through and finish another day. The Progress Checklist needs to perform these major functions:

  • List every inspection in the sim, so that the learner can remind themselves of the inspections that are required.

  • Show if the inspections are completed or not, so that the student can quickly see if they have forgotten to inspect anything.

The inspectables are broken down into categories according to where they are located on the truck. Each “tab” of the checklist contains all of the inspections in that POI. NOTE that these tabs are not required to be a 1:1 match for the POI menu in the Adv mode UI, but that would be a nice bonus.

The ProgressChecklist class handles the following:

  • Toggle open/ close the checklist

  • Update elements status

  • Calculate the remaining tasks

  • Update tabs status

  • Trigger save used for Save and Load feature

There are 2 main classes that run this system:

  • ProgressChecklist

  • ChecklistTab

 

Progress Checklist

Progress Checklist Options

User Interface

  1. To toggle the visibility of the progress checklist from the Checklist button in the bottom left toolbar, call ToggleOpen(false).

    This tab shows that some of the inspectables at the POI “DS Cab Exterior” have been completed, and the whole progress is 8 of 232 inspectables was done.


2. To toggle the visibility of the progress checklist from the Finish button in the bottom left toolbar, call ToggleOpen(true), there will be a prompt to remind you to review the checklist before submitting the results.

Checklist Tab

A checklist tab has a title, an outline (color to be changed) and a background (color to be changed).

UpdateColor(ProgressStatus) method is to update the tab’s color accordingly:

  • ProgressStatus.notStart: background and outline color of the tab are gray

  • ProgressStatus.selected: background is gray, outline color is yellow to highlight the tab which is selected. By default, every time toggle open the checklist, the 1st tab is always opened and selected (highlighted).

Final design is that the tab only changes border color once it’s selected, no color changed for each tab if it’s been completed, or all the tabs if the full inspections list is done. 

Setting up the inspectables for the ProgressChecklist

Each inspectable will have a Position number starting from 1 which is used to generate the game object for the tab of that Position number (Advanced Mode Position is obsolete).

There are 11 tabs in total (Position 1 to 11).

Setting up the ProgressChecklist with the Advanced Mode

Here is the design of the Checklist to work with the Advanced Mode:

  1. Visual inspections (no physical inspection button to click):

  • As soon as the learner enters a given POI, all of the visual inspection tasks that are enabled in that POI will be marked as completed.

For AdvancedModeProgressChecklistMenu script:

  • In each view menu button (ie: DS Cab Exterior button of Advanced Mode Buttons prefab), put all the inspectables of that POI to the list “Inspectables Game Objects”. These inspectables will be marked as inspected once the button is clicked.

  • For the lights inspection, put all the lights of that POI to the list “Lights Game Objects”, these lights will be marked as inspected only after turning on the lights in the Cab Interior position.

  • For warm inspection, in this example, it’s Driver Side Engine position, put all the inspectables that require warm check in the “Engine Warm Game Objects” (DS Wiring and Engine Air Hoses), these inspectable are marked as inspected only after turning on the engine in the Cab Interior position.

 

2. Physical inspections (requires a physical inspection button in the left vertical tray):

  • As soon as the learner clicks a physical inspection button, that inspectable will be marked as completed.

For AdvancedModeProgressChecklist script:

  • For each physical inspection button (ie: Engine Oil Button), put its inspectable in the list “Inspectable Game Objects”, this inspectable will be marked as inspected once button is clicked.

  • For inspectables that are side sensitive (using the same physical button, ie: ubolts, hood mirrors, door mirrors, …), put the driver side inspectable to the 1st element of the list “Inspectables Side Sensitive”, and the passenger side inspectable to the 2nd element of that list, those 2 inspectables will be marked as inspected once button is clicked, according to the side.

3. Nested physical inspections (requires initial click on a physical inspection button, which will then bring up a subset of physical inspection buttons inside the specialized view, e.g. Wheels):

  • As soon as the learner clicks on the first physical inspection button (e.g. Wheels) the inspections that are visible in that specialized view (e.g. sidewall and rim) will be marked as completed.

For AdvancedModeProgressChecklist script:

  • For the main Wheel inspection button, simply put that button in the “Advanced Mode Wheel Button”, this will get the CurrentWheelController and all of its inspectable elements, these will be marked as inspected once the main Wheel button is clicked. These elements are:

o   Side wall

o   Lug nuts

o   Tire tread

o   Tire pressure

o   Brake chamber

o   Brake drum

o   Axle leak

o   Brake air hoses

o   Shock

o   Air bag
