VARLab : Adding Video Conditions to Inspection Windows

Created by Patrick Cho (Unlicensed) on Dec 12, 2022

Introduction

This Confluence page will explain how to add videos instead of images to inspection windows.

Preparation

First, you will need to get your video clip downloaded. If you don’t have a video clip, and only have a gif available, just use a free online tool such as https://ezgif.com/gif-to-mp4 to convert it. In most cases, the video size will be larger than we want it, so you want to encode the video using https://handbrake.fr/. This will significantly reduce the size of the video, which is important not only for reducing the size of our sim, but improving loading speed as well. Additionally, videos over 30mb are not able to be included in WebGL builds under our build configuration. Once your video is downloaded, name it following the Heavy Equipment naming convention of Component - Condition (i.e. WorkLights-Flickering). Next, add it to the Assets/StreamingAssets folder in the Heavy Equipment project. This is the required location that the code backend is looking for the videos when passed a string filename.

Implementation

To add videos to inspection windows in place of images, there are two changes you need to make in the Specialized Inspection Condition you are wanting to use a video for.

First, set the “Media” enum field to type “Video” (as opposed to image), and then set the Video Name field to the name of the video file (including file extension).

IMPORTANT: Always remember to add the file extension to the URL field. Otherwise, it won’t be successful in finding the video. Additionally, the term “URL” is a bit confusing, as you shouldn’t put an actual URL link in here, but rather the video name with extension. This is just what Unity’s video player refers to that field as.

If the video is placed in the correct folder, the “Media” dropdown is set correctly and the video name exactly matches the file name of the video, the video should show up in your inspection window without any issue.

If it does not show up, ensure that the video is not super large, that your “Video Name” field and file name match exactly, the enum is set correctly, and the video is in the correct folder.
