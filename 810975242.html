Our Persona

Who

A first year student in the Paramedic Program

Our learner is interested in being a Paramedic as a profession in the future. And learns through hands-on training for a better grasp in knowledge.

Goals

To be prepared and understand the paramedic equipment utilized in managing patients in the prehospital field.

Technology Pre-requisites

Program device requirements - your program | Conestoga College

HMW (How Might We…)

Enhance the digital learning experience of the paramedic student with no prior knowledge, to find and recognize all of the equipment, compartments and supplies available in the Ambulance.

Challenges

Not all students can be given an in-person demonstration of an Ambulance in addition to catering to the diverse learning styles of the students.

Scenario 1: Explore exterior compartments of the Ambulance

Expand to view - Scenario 1: Explore exterior compartments of the Ambulance

Brief Description

Learner starts to explore the exterior compartments of the Ambulance, in order to get familiar with the equipment in each of the compartments.

  • The order in which they choose to explore is entirely up to them.

  • Each compartment contains different equipment.

  • The learner is encouraged to take their time and thoroughly understand each equipment by interacting with it.

  • The learner continues exploring the exterior compartments until they are confident that they have learned about every piece of equipment.

  • They are provided with a checklist in order to show progress in their learning about each equipment examined.

  • The valuable knowledge gained by examining every equipment is reflected on the completion of the checklist.

Description

The student can hear and view the text being read in order to follow along. They can pause it where the highlighted text also stops.

To re-read the text, the voiceover has to be completed on order to play again

Acceptance Criteria:

  • Play and pause buttons on hover change color to show focus.

  • The voiceover plays on auto-play

  • The text is highlighted on the word being read

  • On pause, the voiceover pauses with the current text highlighted

  • The play and pause buttons are interchangeable.

  • Once the whole description is read, then the pause button is replaced with play to allow user to play the text again.

Item Found

As a learner, I want to be able to get feedback on finding an item, so that I know I have found the correct object from the list

User tasks

Learner can select an item from the environment

Learner can view a green highlight around the correct object

Learner can hear a sound that reflects an item found

Learner can see a success message on the screen

Learner can see the item checked with green color status in the list

Learner cannot click on the hint icon button

Acceptance Criteria:

  • On finding an object, the object gets highlighted green

  • Green success message on the screen that appears for 1 second ( user does not have to dismiss it)

  • SFX feedback on success

  • The item in the list gets checked and green font color

  • Hint icon gets disabled

Item NOT found

As a learner, I want to be able to get feedback on finding the wrong item, so that I know I have to keep searching further

User tasks

Learner can select an item from the environment

Learner can view a red highlight around the wrong object

Learner can hear a sound that reflects an item NOT found.

Learner can see an error message on the screen indicating that it is not found

Acceptance Criteria:

  • Object selected and NOT from the list gets highlighted RED

  • Error message on the screen that appears for 1 second ( user does not have to dismiss it)

  • SFX feedback on error

  • Only one error message will be shown at a time

Hints

As a learner, I want to be able to get hints, so that I can get the help when I cannot find an item

User tasks

Learner can click on hint icon for an item

Learner will view which hint has been clicked

Learner can view the hint at the bottom of the screen

Learner can dismiss the hint provided at the bottom of the screen

Learner can ask for hints for the items that have not yet been found

Acceptance Criteria:

  • Hint for the respective item will be displayed at the bottom screen - when prompted by the user

  • Hint icon changes color when clicked

  • Hint icon for items Found will be disabled

  • Hint icon for items NOT found will be enabled

Keyboard and Mouse Interactions

Using Keyboard Interactions

  • When using keyboard navigation, the following keys are fundamental:

    • TAB: Navigate forward

    • SHIFT+TAB: Navigate backward

    • SPACE: Check or uncheck boxes

    • ENTER: Activate a link

    • Left, right, up, and down arrow keys: Navigate between or to options

Using Mouse Interactions

  • Left-mouse click for selection

  • Mouse wheel to zoom in and zoom out
