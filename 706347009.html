(PoV) (Assets TBD)

The objective of this DLX is to provide a virtual practice workspace for students to apply what they have learned about electrical circuits and safe work habits so they can apply this knowledge to their in-person skills labs. For each of these modules, there will be lecture, followed approx 1 week later by the in-person skills lab related to that lecture. Students are expected to use the 6 days between the lecture and the skills lab to practice the project and gain confidence. Two projects are planned for this DLX:

  • Build & Test Residential Circuit

  • Commercial Manufacturing Machine Control Panel

  • Conversations with VLT have helped us to arrive at some difficult compromises in hopes of delivering this DLX on time:

    • Many animations will be very simplified or cut entirely. The animations were intended to provide some passive learning value by showing a brief example of how the manual/tangible work would look. We currently have a placeholder animation for mounting boxes that is intended to enhance immersion and possibly act as a reminder of the types of tools and techniques that are used for this work at the job site. The fact is that we will not have time to do most of these animations, and they are not providing any significant learning value. Students need to learn hands-on skills in the skills lab (how to use a drill, how to run cables, how to terminate conductors), not in the DLX. The highest-value animations can be prioritized if we have time for them, but all other animations will be cut from scope. We can always come back and uplift the project in a future project.

    • All existing game mechanics and UI elements must be re-used in their existing form, without further optimization or customization. This could result in some limitations for future workflows such as Select & Install Devices, particularly for outliers like the Exhaust Fan or the Breaker, but we will need to accept those limits and make it work without introducing any new or adapted interactions, and without relying on animations to fill in the gaps. This was mostly already what was planned for the upcoming feature implementations, so it’s really just a good reminder to resist the urge to tinker or optimize.

    • The biggest ingredient in ending the do-redo-redo again cycle that we are stuck in is going to be reducing opportunities for change. Our CE is excellent and wants the very best for this DLX, but he has had a tough time settling down on one right answer in many of our workflows. In many cases, even though the existing workflow is correct and works, he requests an even better version - more realistic lighting fixture, different order of terminating conductors in the gang box for rough-in. Stacey is going to work directly with this CE to minimize temptation to revise and optimize task options or order of steps, unless a critical mistake has been made and the workflow is WRONG. This is a big challenge for us, because our usual practice is to follow the lead of the CE and implement what they ask for. Due to project constraints, we need to be able to push back and respectfully decline any new requests from our CE on this project, unless they are critical bugs.

    • Collect feedback without adding scope. All bugs reported in playtesting or focus group testing will go to the backlog for consideration in a future project, unless they are critical bugs. Stacey will review critical bugs with VLT to determine priority, nothing goes into scope without review.

    • Starting from Sprint 24F5, we will deprioritize further refactoring, technical redesign, or code health requirements in favour of user-facing features and workflows. In spite of knowing how valuable long-term code health and systems are, we have exhausted the amount of time that we can invest in this project for those valid goals. We are trying to strike a balance between a stable, serviceable product and delivering the product as per our commitment. We need to be able to apply our best coding practices moving forward, and not lose any ground.

  • With these directions in mind, the term goal for this project is to complete the full project workflow (including save & load, settings, feedback, all user-facing features) by the end of 24F7 (Dec 06). During the intersession, we can address any lingering concerns about code health or product quality, then attack the remaining workflow steps for Final inspection and all remaining GR requirements (Analytics, Promo video, etc) in the 25W term. Depending on how quickly we can implement those requirements, we can prioritize bugs in the backlog and other high value user-facing enhancements such as animations as time & resources permit.

  • Meeting with Katrina and Sana, and here are the decisions made:

    • There will be 2 short multiple choice quizzes for the student to complete, like knowledge checks at the start of tasks. In each case, the student will get immediate feedback using the existing UI design for Select Supply Cable Type.     • Quizzes only need to be completed ONCE, not once per box. After the knowledge check questions have been completed successfully, they will not be shown again, even if the learner returns to that task at a later point.

16 Jul 2024

Lots of good decisions at the TE Design Sync meeting today! Here are some highlights (these will go into Jira and Figma):

Run Supply Cable Task

  • Learner will only be asked ONCE to select the type of supply cable they want to use.  Not once per box.

  • Immediate feedback will be shown, but we need to change this so that feedback is only shown for the most recently selected option.  As soon as the learner selects a different option, the original selected option should be reset to default appearance.

  • Katrina Fraser is going to decide if Select Supply Cable will be its own subtask in the Run Supply Cable task, or perhaps a separate tab (first on the left?) in the Run Supply Cable task menu.  Works either way.

Secure Cables Task

  • Just like selecting the supply cable type, this is a task that will only be done ONCE, not once per box.

  • This can be done as a quiz, in its own panel.  It can also be done in the option selection menu. Katrina Fraser to decide.

  • Katrina Fraser will decide if we want the learner to individually select the cut length, unsheath length, and securement position, or if these should be combined into ONE decision similar to the Run Supply Cable task (learner needs to consider multiple factors to make a decision).

  • Feedback for the quiz or the selected option(s) can be provided immediately, using the same type of Supervisor Feedback that was used for Select & Mount Boxes.  Otherwise, feedback can be provided in the Rough-In Inspection results, either as CODE feedback in the inspection report (we have not yet identified a lot of CEC feedback for securing cables, but we might still get more input from Tom) or as Project Feedback from the supervisor.  Katrina Fraser to decide when and where feedback should be shown.

  • Revising the Secure Cables task can work just like the revision for the Run Supply Cables task:  Learner can initiate the revision by selecting the completed task in the task menu OR by clicking in the scene:

    • Click on the supply cable in the scene - go to Run Supply Cable option menu in the task menu (no options are disabled, you may select any option including the currently selected option)

    • Click on any box in the scene - go to Secure Cables quiz or option menu (no options are disabled)

  • Because there are only 3 options for Run Supply Cable and Secure Cables tasks, we will NOT reduce the number of options even after feedback has been given.  After feedback has been given, only the wrong option that was currently selected is disabled - the other two options will be available.  This must be refreshed after every feedback, so that ONLY the currently selected option is disabled after every feedback.

Terminate Wires Tasks

  • This task will be done once for each box.  The task will follow the familiar workflow of Select task → Select object → Select location → Preview → Confirm.

  • Note that the Gang Box has 4 cables, so the student will have more wires to terminate (12 instead of 3) in that box, but it is all just one task.  The learner will initiate this task in the task menu by selecting Terminate Supply Cables and then selecting Terminate Wires sub-task, then selecting the box they want to work in.

  • When the learner selects a box where they want to terminate wires, they will be taken directly to a special close-up view of that box.  The supply cable will already be cut, unsheathed, and secured in the box. The 3 wires will be slightly spread out so that they are easy to select individually. For the Gang Box, we should consider working with just one cable at a time, and having other cables appear ghostly/muted to prevent confusion.

  • The learner must terminate wires in this specific order : Bond, Neutral, Hot.  If they select the wrong wire, immediate feedback must be shown to redirect them to try again. They must not be allowed to proceed with terminating wires in the wrong order.

  • Learner selects a wire by clicking on it - wire must highlight on hover highlight and when selected.  As soon as a wire is selected, the location options for that wire are displayed, using tiny floating orbs that are similar to the location option markers used for Select & Mount Boxes workflow.  Location options will vary per wire, but options will include the following types of termination points:  bond screw, new (empty) marrette connector, add to existing marrette connector, leave it sticking loose out of the box, tuck neatly into the box.  Orbs will need to have hover tooltips to help describe the option that each one represents. 

  • When the learner clicks on a location option, they see a preview of the selected wire connected to the selected location.  No animation is required for the preview - it just magically snaps into place.  Learner can Confirm or choose again.

  • When confirmed, a brief animation shows the wire being terminated according to the selected option.

  • If the learner selects a wire that has previously been connected, the task is undone and they need to select a location option for that wire. No options are disabled, they can select any option for that wire including the previous selected option.

27 Jun 2024

  • A new interaction for wire connection workflows (e.g. Terminating wires in boxes, installing devices in boxes) was proposed. Instead of selecting wire placement from 2D images in the task menu, the student will work in the 3D scene, in a special close-up view at each box. The primary objective for this change is to get the student doing more of their work in the 3D scene instead of being stuck in the 2D menu all the time.

  • The click-connect interaction will be similar to the one used in MPC, where the student would select a lead and place it on the trainer board by selecting the connection ports. In TE, there is no board, so we will provide connection options using the familiar yellow orb that was used to mark locations for mounting boxes in the Select & Mount Boxes workflow. Also, in MPC the learner had to connect both ends of the lead to the trainer board, but in TE there is only 1 end of the wire that needs to be connected.

  • To complete each task, the learner will select a wire (wire will highlight) to see connection options for that wire. Connection options are described in this document TE Scenario 1 – Supply Cable Task Options and Feedback.docx in the Terminate Wires in Boxes section. Each wire has approximately 3 options, which are not necessarily unique (multiple wires can be connected to the same point).

  • The learner must select one of the connection options shown to complete the task. The task must be repeated for each wire in that box (in most boxes, there are 3 wires (bond, neutral, hot) but in the Gang box there are 4 cables with 3 wires each, for a total of 12 tasks in that box.

  • There is a correct order for these tasks, which we can enforce by redirecting the learner if they select wires in the incorrect order. This immediate feedback can use the same feedback UI that is used for other tasks (e.g. immediate feedback when selecting supply cable).

  • Unlike MPC, we do not need the wire to have any physics - it will not follow the mouse cursor to “stretch” to the selected destination. After the learner selects a destination for the selected wire, a brief animation can show that action.

Objectives for Trades - Electrical

The objective of this DLX is to provide a virtual practice workspace for students to apply what they have learned about electrical circuits and safe work habits.

Link to Project Charter in SharePoint: Project Charter - Electrical (RF).pdf

The project charter describes 2 scenarios the DLX should cover:

  1. Build & Test Residential Circuit - The goal of this lab is for the student to successfully install and test a basic circuit in an unfinished wall (framed, no drywall) of a residential building. The main tasks for this module would include the following key decisions:

    1. Review the project plan and schematic circuit diagram to understand the success criteria for this electrical project

    2. Select the proper equipment from a selection of components (boxes, switches, outlets, etc)

    3. Select the proper location for each item to be mounted: outlet box, gang box, fan box, light box

    4. Select the appropriate cable to run for this project

    5. Select the path for the cable to be installed, including speed loops and fasteners as needed

    6. Install and terminate cable in each box, including removing the outer cable sheath, securing the bond wire, and leaving the correct amount of loose wires protruding from the box.

    7. Select the proper breaker and location to install the breaker in the residential electrical panel

    8. Call for the Rough-In Inspection at the appropriate time - inspection must pass or student must remediate any code violations and repeat the inspection until it does pass.

    9. Select the proper devices from a selection of items (switches, outlets, fan motor, lighting fixtures, etc)

    10. Install each device by connecting each supply wire to each device correctly

    11. test the circuit without power to ensure that everything is properly connected, then turn the power on and test again to make sure everything is nominal.

    12. Call for the Final Inspection at the appropriate time - inspection must pass or student must remediate any code violations and repeat the inspection until it does pass.

    13. Receive project feedback from “homeowner” or “boss” to ensure compliance with project plan specifications

  2. Verify machine has no power - The goal of this lab is for the student to successfully verify that power is cut to a machine control panel in a commercial manufacturing facility setting. For example, the student would need to test and verify that a machine press at a car manufacturing facility has no power and is safe to work on.

Assumptions / Known Constraints

  • Simulations developed under this plan are not intended to be played on tablets or cell phones.

  • Simulations developed on this charter are not guaranteed to have full functionality on devices that do not meet standard program device requirements.

  • Simulations developed under this charter are not intended to replace the teaching of the learning content, but rather augment it. They are intended to be part of formative learning experiences, not formal assessments. The simulation will not report any grade to the LMS, or provide any lasting record of the student’s performance outside of the current user session.

  • These simulations will not be designed to play with VR headsets or controllers.

  • These will not be multiplayer simulations (i.e. students playing in pairs or the professor having the opportunity to be in the simulation with the student at the same time).

  • Completion status will be recorded ONLY for the currently logged-in student in the LMS. If students work in pairs at a shared computer, only one of them will show as having completed the simulation.
