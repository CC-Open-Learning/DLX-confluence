VARLab : Bulldozer - Adding Analytics To A Heavy Equipment DLX

Created by Brodin Collins-Robb, last modified on Aug 08, 2023

This page explains how to add analytics to specifically heavy equipment DLX projects. This page builds off the analytics pages generally set up by the analytics team. In HE project analytics will add info capturing about:

  • Different users playing the simulation both in Unity and in a browser

  • Inspection completions

  • DLX completion

 Instructions

1

Ensure the CORE HE Inspections package is on version 1.4.1 or later

This is to ensure that the fileName variable we will use for user analytics is accessible as a public variable in the scenarioSaveHandler.cs script

2

Install PlayFab package

Find instructions here: https://learn.microsoft.com/en-us/gaming/playfab/sdks/unity3d/installing-unity3d-sdk

3

Add the analytics package to the project in the package manager

In the package manager, add a package by git URL:

ssh://git@bitbucket.org/VARLab/analyticspackage.git#upm

Note: Above steps copied from: Installation of the Analytic Package

Add the analytics object to the scene

5

Created and set up an equipment-specific analytics script - i.e., bulldozerAnalytics

Add a new script in the below location:

The script should follow along the lines of the below code:

using System.Collections;
using System.Collections.Generic;
using System.Diagnostics;
using UnityEngine;
using VARLab.Analytics;

public class BulldozerAnalytics : MonoBehaviour
{
    //this channel allows us to send or connect to our playfab account
    public AnalyticsEvent analyticsEvent;

    //Data to send to anayltics
    public int numInspectionsCompleted;

    public void setNumInspectionCompleted(int numComplete)
    {
        numInspectionsCompleted = numComplete;
    }

    //This method will send an integer (num of inspections completed) to the analytics channel
    public void SegmentCompleteMethod()
    {
        //send integer
        SegmentCompleteContainer data = new SegmentCompleteContainer("InspectionsComplete", numInspectionsCompleted);
        analyticsEvent.Raise(this, data);
    }

    //This method will send a boolean value of when the simulation is complete (simple true value sent to the analytics channel)
    public void DLXCompleteMethod()
    {
        //send that we are done
        DLXCompleteContainer data = new DLXCompleteContainer();
        analyticsEvent.Raise(this, data);
    }
}

8

Add a getScormFileName script

Add a new script in the below location:

The script should follow along the lines of the below code:

using System.Collections;
using System.Collections.Generic;
using UnityEngine;

namespace HeavyEquipment
{
    public class GetScormFileName : MonoBehaviour
    {
        public ScenarioSaveHandler saveHandler;

        public string FileName { get => saveHandler.fileName; }

    }
}

9

In the FSM - Add three blackboard variables to the FSM (Ensure the variables are added in both the prefab FSM and FSM

Using the add variable button at the top of the backboard variables [LEAVE VALUES BLANK FOR NOW]:

  1. add a “game object” for our analytics object

  1. add an “integer” for inspections complete count (will be self-bound to saveData inspections completed variable)

Using the add variable button in the middle of the backboard variables:

  1. Add a “string” for our fileName (Self-bound to getScormFileName script)

10

Add scripts into FSM under Prefs

In the FSM, click on Prefs on the top left and follow t through the menus below:

The below menu will appear; click on add new type:

Then the below menu will appear where you will search for your scripts and click them to add them:

You will need to add these three scripts:

  • COREanalytics

  • getScormFileName

  • equipment-specific script (i.e., bulldozerAnalytis)

11

In the FSM, add values for the three variables

  1. Analytics - drag the game object from the scene into the value of this blackboard variable:

  1. insepctionsComplete - click the cog on the right of this blackboard variable and follow through the menus shown below:

  1. fileName - click the cog on the right of this blackboard variable and follow through the menus shown below:

12

Add the login analytics functionality

Under the “Load Saved State” node in the FSM, add a new execute function action task on this node

Set the analytics object in the analytics function - ensure to check off the check box and the dot:

Call the loginLearner method from in the COREanalytics scripts

Set the parameter “Learner ID” to the fileName blackboard variable:

13

Add inspection complete analytics functionality

Under the “Update Saved Data” node, added a new execute function action task

Use the analytics game object blackboard variable (Ensure to check off the checkbox and click the dot)

Call the setNumInspetionCompleted in the bulldozerAnalyitcs scripts

Pass the parameter insepctionsComplete from the blackboard variables

Then add another execute function action task to the “Update Save Data” node in the FSM

This action task uses the analytics blackboard variable (Ensure to check off the checkbox and click the dot)

Call the segmentCompleteMethod in the bulldozerAnalytics script

14

Add DLX end analytics functionality

Added a new execute function action task to the “Inspection End” node in the FSM

This task uses the analytics blackboard variable (Ensure to check off the checkbox and click the dot)

Call the DLXCompltete method in the bulldozerAnalytics script

15

Seriliaze last script

Add the getScormFileName script to the bulldozer state machine and then serialize the cloud save handler from the scene to the save handler variable of this added script:
