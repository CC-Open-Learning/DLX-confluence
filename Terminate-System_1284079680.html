VARLab : Terminate System

Created by User c4ba3 on Aug 15, 2024


Current Issues:

  1. Exiting a task is not possible.

  2. Switching wires and altering options is not feasible.

  3. The implementation of termination status types is pending.

  4. Each subtask requires a simple enumeration status, which should be updated when an orb is selected.

File Descriptions:

  • Terminate Cable Task: This file encompasses references for subtasks within the wire categories (bond, neutral, hot), alongside task view camera settings and the main task functionality. It primarily facilitates interaction with subtasks, enabling orbs, the termination option, and removal.

  • WireSubTask: Represents the wire object.

  • TerminateWireAnchor: A script placed on the final wire object to display it to the user upon clicking a terminate location orb. It tracks the selected wire, transferring data to the anchor point when the original wire is disabled, as it includes a WireSubTask. This feature could be integrated into the TerminateLocation script but needs to be checked more.

  • TerminateCableLocation: Script linked to the orb, referencing the game object it will enable, with the TerminateWireAnchor attached to that position.

  • TerminateCableTaskHandler: Identifies all objects related to the TerminateCableTask, organizing them in a dictionary by their task for reference for handling purposes. This script contains the methods required to be called for starting, setting up orbs, and termination.

  • TerminateInteraction: Manages raycasting to determine interactions with wires or orbs.

  • TerminateEditInteraction: Handles raycasting and determines interactions with already terminated wires.

System Functionality

The user engages with the Terminate Cable Task handler by initiating the selected task. A new task is created in the Task.cs file, serialized in TerminateCableTask.cs, and attached to an empty object. The task name enum passed is used to access the Terminate Cable Task.cs file, activating the camera and recording the task initiation in the backend.

Post-camera preview, the cable wire colliders should activate (currently, only the object is enabled). The user can then select a cable, revealing the orbs. Upon orb selection, the finalized version is triggered by activating the object linked to the orb's serialized location.

The user can now edit the terminated wire by clicking on it, causing the finalized wire to disappear. This action accesses the orb locations for that wire, shows them, and allows the user to reselect the option.
