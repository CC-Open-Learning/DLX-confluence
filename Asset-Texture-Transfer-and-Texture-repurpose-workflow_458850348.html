VARLab : Asset Texture Transfer and Texture repurpose workflow

Created by Rounuk Kumaran (Unlicensed), last modified on Nov 11, 2021

For the Trucking Sim, we have an acquired 3D model of the Truck that has its own UV layouts and Textures already available. Our goal is to use these models to get our high Poly and Low Polyrestructured.

We can transfer the existing textures of these HighPoly models onto the Low Poly model that has its UVs updated. For this demonstration, I am going to use an example to simplify what it means.

UNDERSTANDING THE POLYGON AND ITS UVS

For example, here we have a Tire that was taken from the Existing Truck model

The Model has a higher number of polygons and has its existing Uv setup and textures accordingly.

What we now need to do is convert the high Poly to a Low poly and change its UVS to a better Layout, or the layout we need. For this demo, I have just unwrapped the Low UVS to the whole UV page.

Our goal is to use the Highpoly above and its textures to transfer the texture details to the low poly according to its UVS. It can be done in very simple steps using the XNormal software which is free to download.

XNormals download Link: https://xnormal.net/

Firstly for our project we are Using the PBR (Metallic/Roughness) workflow. So we need Albedo maps and Normal maps transferred from the existing mesh to the new mesh.

What we have is the Bump map, we need to convert it to a Normal map, this can be done in XNormals.

What is a Bump Map

A bump map is a Black and white Map and is somewhat similar to a Displacement map to look like, but both maps fill different requirements.

In a Bump map, the whites represent raised platform and the blacks represent depth.

While transferring this Bump map into Normal Map, The software uses the brightness/darkness values to determine how to translate bumps and depths in the normal map.

Not all bump maps have good brightness values so we need to double-check that and increase the brightness values wherever necessary.

Bump Map to Normal Map

In XNormals go to the tools options.

Of the multiple options choose the HeightMap to Normal Map

This will open a window where we can plug in the Bump map and Generate a normal Map, The window looks like this:

Initially, both slots look empty. We need a couple of steps to create the map:

  • Drag and drop your bump Map texture on the Left-hand side of the two panels.

  • Right mouse click on the empty right slot, amongst the options click Generate.

  • Right mouse click on the now generated Normal Map on the right slot and click save Normal Map.

This makes the Normal Map to be exported from XNormals to the downloads folder.

The Normal maps created might not have the background Flat colors sometimes, so we need to go to a photo editing software and add the color to it. The flat color for Normal Map has the Hex Code: 8080ff, or it can be created by adding RGB values: R=128, G=128, B=255.

Transferring Albedo/Normal Map from High to Low

This method was discovered from a post. Having watched the overview tab the next post makes more sense.

Thus we are using the same idea for both Normal Maps and Albedo map transfers.

In the Highpoly window, add the Highpoly object with its Original UVs in the file tab

In the base texture to bake tab, we will add the Original Albedo or Normal map for the high poly.

In the next box, depending on what map we plan on transfer we check or uncheck it. The box has to be checked only if we are transferring the Normal Map. We shall now add the Low Poly

For the Low poly stage, we just need to drag and drop the low poly just like in the photo below, we can also right-mouse click on the File tab and click add mesh… similarly to adding HighPoly mesh in the Highpoly window. After that, we go to the baking options.

The final step is to hit the Generate Map button on the Bottom Right Corner of the software to
