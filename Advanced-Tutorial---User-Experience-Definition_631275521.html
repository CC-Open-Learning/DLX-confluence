VARLab : Advanced Tutorial - User Experience Definition

Created by Stacey Dineen (Unlicensed), last modified on Mar 20, 2023

This page will define the user experience for the Tutorial inside Advanced Inspection mode.

Launch

After selecting Advanced Inspection from the main menu, the learner will be welcomed to the Advanced Inspection. To access the Tutorial for Advanced Inspection, the learner will click on the Tutorial button in the main toolbar.

Initial View and Welcome

On initial load, the How to Play menu will be shown. The voiceover will welcome the student and prompt them to select a topic that they want to explore. Note that the Tutorial Menu from Guided Inspection does not apply here, because there are no Truck Overview or Walkthrough Videos for Advanced Inspection. This basically means that the Advanced Tutorial launches straight into the How To Play area.

How To Play - Segmented Help for How to Complete In-Game Tasks

In Advanced Inspection, the learner can select from the following short menu (could be pictorial menu, for a consistent look & feel with Guided Tutorial, if that works) of How To Play topics:

How To Play will NOT be interactive in the Advanced Tutorial, like it is in Guided Tutorial. Each topic will be a video walkthrough with voiceover. Sound effects and voiceover from the sim will be suppressed to avoid interference with the tutorial voiceover.

Proposed Topics for How To Play - Advanced

Navigate using the View Menu

  • Show location of the View menu

  • Select a location to inspect - these can be selected in any order, so select something other than the first location listed, just to demonstrate that they are available

  • Select a different location - there is no need to go in any specific order, you can return to any location as many times as you want.

Get a Better Look

  • Zoom the camera

  • Pan the camera

  • Move - use Move arrows in the UI, but also mention keyboard arrows and A & D keys.

  • Explain that you can reset the camera to the default position by selecting that POI from the View menu again.

Report a Problem

  • Click on the item that has a problem.

  • Click on any answer (note there is no feedback at this point)

  • Remind the student that they will not get feedback about their inspections until the end of the learning experience.

“Locked” Inspection Tasks

  • Some inspection tasks can only be completed when the engine is cold (turned off) or warm (engine running). If you try to do these inspections at the wrong time, they will be locked.

  • Show an inspection component such as fan/fan belt that cannot be clicked (tooltip is shown) when the engine is running. Tooltip says “Engine must be turned off”

  • Show a physical inspection button that is locked, such as engine oil pressure gauge on the dashboard (tooltip is shown) when the engine is not running. Tooltip says “Engine must be running”

Inspection Checklist

  • Click on the Checklist button in the main toolbar

  • Review the sections, how to tell if something is Completed or not

  • Use the Checklist to remember all of the inspections that are required, since there are no prompts to guide you.

  • Checklist does not indicate if your inspection results are correct or incorrect - it only helps you keep track of where you’ve been and where you haven’t inspected yet.

Results Panel

  • How to complete the inspection (show the button location, inside the Checklist UI)

  • Show Results Panel, how to understand results (correct answer, selected answer)

  • Mention that only incorrect answers are shown in Results Panel so that you can focus on areas you need to work on.

  • Remind the learner that instructors do not see results, this experience is not graded.

Learning Objectives and Grades

  • Objectives are covered in the intro.

  • “Not graded” is covered in the Results topic.
