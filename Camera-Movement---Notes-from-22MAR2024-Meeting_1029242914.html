VARLab : Camera Movement - Notes from 22MAR2024 Meeting

Created by Stacey Dineen (Unlicensed) on Mar 22, 2024

In the 15MAR2024 playable demo, the CE noted that it was difficult to see/identify all of the location options available for mounting some of the boxes, because the user does not know where to look, and some of the options are a little bit hidden due to the default camera position.

The CE also found that measurements were very hard to see/read, because they are shown only on hover, and they may not be facing the default camera position.

The team had a meeting on 22MAR2024 to discuss the camera movement, in consideration of the following requirements:

  • It must be reasonably easy for the student to see all of the location options that are available for the box they have selected to install, even if they don’t know where to look

  • It must be reasonably easy to see and read the measurements that are relevant to the mounting location for the selected box.

  • Camera movement must be optimized to prevent user discomfort. As little movement as possible, using as few clicks as possible, so that users do not feel disoriented or nauseous.

  • Deliberate movement is usually preferable to unexpected movement, but we want to be economical and user-friendly in the number of clicks required to achieve a task. When camera control becomes too manual (not automatic), users become frustrated and are distracted from their learning activities. We want learners to focus on their electrical circuit, NOT on the camera controls.

At the meeting, a number of suggestions were shared and discussed. This page does not seek to capture the entire discussion, but instead this info will focus on decisions that were made by the team.

Team members present: Amila, Colm, Katrina, Stacey, Sana, Vince, Ali

Decision #1: We need to be careful about the default view and zoom level, to ensure that we are providing an optimal initial camera position for each task. We all acknowledged that different tasks are going to require different views, and it isn’t going to be possible to have one fixed view that would work for all tasks. The default view for each task will be chosen carefully so that the learner can immediately see some areas of interest before they begin looking around.

Decision #2: We need some on-screen prompt to make sure the students are aware that there are more location options available, if they pan the camera left, right, up, or down. We will be careful to ensure that the prompt is not directive so that students will not perceive it as a requirement or a hint about the correct location. Mockup was shared like this:

Decision #3: We need some user-controlled navigation in the bathroom scene, so that the student can “walk” from one end of the room to the other. We will start with two navigation targets, which will appear like targets on the floor, one at each end of the room, approximately like this:

This will allow the learner to manually “walk” to either end of the bathroom to get a different vantage point when selecting a location to mount different boxes.

Decision #4: When a learner selects a location option, we will automatically show an optimal view of that location, including automatic adjustments to pan, zoom, and camera position as needed. When a learner selects a location option, we will automatically navigate to the closest camera position (navigation target) for that location option. This will minimise the number of clicks required to preview a location option, by always navigating to the optimal camera position for each selected location.

Decision #5: In the current demo build, the preview action (view the box in the selected position, with measurements shown) is achieved by hovering the mouse over a location option. This could result in a lot of unintended camera movement because it is so easy to accidentally hover your mouse over a location option. We decided to create a distinct left-click action to Preview a location option, so that the camera movement would be deliberate. The workflow would need to work something like this:

  • Left-click to ‘select’ a location option for Preview: Camera movement happens automatically. Box is shown in semi-transparent style (indicating preview, not installed yet) in that position, with relevant measurements visible and readable.

  • From this Preview state, the learner can confirm that they want to install the box at this location, or they can pan the camera or manually navigate to select a new location to Preview. If a new location is selected with left-click, the camera once again automatically moves and the preview is displayed for the new selected location.

  • When the learner confirms their selected location, they will see the animation of the box being mounted at the selected location. The box will stop being semi-transparent and will appear in the scene normally. All other location options will disappear.

Decision #6: The learner must always be able to see the essential information (box and relevant measurements) throughout the workflow. We will facilitate this by allowing pan and zoom at both of the two camera positions (navigation targets) in the bathroom. We will carefully select min and max zoom levels that are appropriate to each view, and we may set pan boundaries as well where appropriate to help ensure that learners maintain a good view of the work area.

Decision #7: Measurement values are not as useful when the learner can’t see both ends of the measurement on the screen (e.g. if they zoom in, they will only be able to see part of the measurement line, so they might lose track of the two ends points for the measurement value. We will always try to keep the measurement value visible on the screen, perhaps using “floating” style of display similar to a tooltip, but always keeping the measurement value close to and clearly associated with the measurement line. We decided to add text to the measurement value to identify what we are measuring (e.g. to floor, to vanity, from bathtub) to help remind learners of the end point of a measurement without requiring them to zoom all the way out to see it.
