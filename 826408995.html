VARLab : Proposed Scenarios for PHI (aka bad state sets)

Created by Stacey Dineen (Unlicensed), last modified by Carolina Naoum Junqueira on Feb 28, 2024

We use ‚Äúbad state sets‚Äù or ‚Äúseeds‚Äù to organize logical groups of inspection results within a DLX user experience. These ‚Äúbad states‚Äù are the reportable problems that the student is looking for when they perform an inspection, so we want to make sure that they are adequately distributed around the game environment and are logically linked where appropriate.

The trick is to create enough different scenarios that the student feels as if the bad states are ‚Äúrandom‚Äù, without actually having them be random. The problem with randomly loading bad states is that they can all clump up in a single location, you can have unrealistically high or low numbers of bad states in a given playthrough, or the bad states can be logically in conflict with one another (one item in the fridge can be too warm, another can be too cold).

The goal is to distribute the bad states in a broad but unpredictable way, and make the experience really engaging and challenging for the student.

Note: We may add more inspection locations and bad states as needed to each scenario as we continue to build the DLX. This is a work in progress.
