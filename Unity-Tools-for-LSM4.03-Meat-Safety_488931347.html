VARLab : Unity Tools for LSM4.03 Meat Safety

Created by David Inglis (Unlicensed), last modified on Jun 01, 2022

Objective

To increase the speed of development for the OMAFRA Meat Safety Digital Learning Simulations, a number of off-the-shelf Unity tools have been purchased an imported into the Unity project space. Descriptions of these tools are provided below along with their intended use-cases and links to documentation and other helpful resources.

Tools

Skybox Blender

Skybox Blender is used to transition between skybox / cubemap assets which consist of 360 photos of the facility. The intended use-case is to develop a “Google StreetView” style interaction where the learner can move between key areas of the facility, using clickable zones (ie. buttons) on screen to navigate.

Transitions between these skyboxes may be filled with pre-recorded video of a person moving through the facility, if this video provides useful context for the learner as they move between key areas.

The Skybox Blender plugin is located in CORE Engine in the Assets/Packages/Skybox Blender and includes the Skybox Blender Demo scene.

Drag & Drop

Drag & Drop Pro provides the functionality to click and drag on UI elements and drag them around the screen with the mouse. They can be dropped on a target area to trigger a new behaviour, such as the completion of a task. If the dragged element is not dropped on the target, it can either be left where it is or reset to is starting position.

Considerations

The documentation for this asset notes that each object and panel you create must have an ID specified before running the application. The documentation fails to indicate that these IDs must be unique, otherwise unexpected behaviours will occur.

Scratch Card

One of the planned interactions for Meat Safety is to have the learner perform step-by-step cleaning procedures for different equipment. Using their mouse cursor and the Scratch Card plugin, learners can scrub up dirty equipment to reveal an image of the soapy equipment beneath it. After fully soaping up the dirty equipment, the next stage would have them reveal cleaned equipment underneath the soapy image.

Actual steps for equipment sanitation will be provided by content experts.

Considerations

As shown in the Scratch Card Demo scene provided by the author, the Scratch Card plugin uses an orthographic camera to position the scratch card on top of the background, as sprite renderers. An orthographic camera works fine for any 2D scene. However, if the scene was set up with a skybox and camera controller allowing the user to pan around, there may be issues with the positioning of the scratch card, and the mouse interaction with the scratch card, if the camera has been rotated away from the default orientation.

If using this asset in conjunction with other interactions in the same scenario/scene, keep in mind that the default position of the camera should be restored before enabling the Scratch Card functionality. Transitioning the camera between perspective and orthographic may be necessary depending on the use-case.

TopDown Engine

TopDown Engine allows for quick and easy creation of 2D top-down games. The use-case for the Meat Safety DLS consists of creating a 2-dimensional map of the production facility, and requiring learners to navigate the facility as different employee roles, ensuring they take the correct paths through the facility, avoiding any “blackout” zones, where certain employee roles are barred from entering for safety and sanitation reasons.

This top-down 2D game interaction uses many Unity first principles - keyboard/button input, sprites, colliders, rigidbodies, and triggered events. TopDown Engine provides an array of tools for rapid development of such an interaction.

Considerations

The TopDown Engine is designed to be added to a brand new Unity project, since it has custom values for many of the Unity Editor’s settings. It should be noted that importing TopDown Engine into the existing CORE Engine project may cause unexpected behaviour with existing DLS Scenarios, since the existing settings have been overwritten by TopDown Engine.

There is currently not a plan to merge the simulations and code developed for LSM4.03 Meat Safety back into the main CORE Engine repository, so unexpected behaviour in the existing DLS project areas is a non-issue.
