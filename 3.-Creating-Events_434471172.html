VARLab : 3. Creating Events

Created by Colm Ambler (Unlicensed), last modified on Sept 13, 2021

Considering Events are more or less RPG Maker’s functions, there are many different things you can perform with them that would lead to this document being overly long. Instead, we’ll go over the basics, like…

Step 1:

Make sure to be in event mode

Step 2:

Double-Click on any tile (you can also right click and press new)

Step 3:

Choose conditions and statements

------------------------------------------------------------------------------------------------------------------------

Event Editor

1

Name and Note

Title the event properly so that you can reference it easier in the event index (located in the same spot as the the tilesets, it changes when in event mode)

The notes, once again, refer mostly to plugins. They vary from plugin to plugin

2

Copying Event Pages

This is fairly self explanatory, but…

New event page creates a new page and adds another number to the event tabs.

Copy event page copies the current event page you’re looking at.

Paste event page pastes in a new tab, not the one you’re currently in.

Delete removes the entire page and tab.

Clear event page removes any event contents but keep the page.

3

Event Tabs

Each event can hold up to 20 different event pages, each getting added to a row of tabs in this spot.

*It is important to note that when the event is triggered and checks the conditions, it goes through all of the pages and chooses the last one in the tab.

*For example, if I had three event pages, one with a variable, another with a switch and a third with an actor condition. As long as the actor condition is fulfilled it will always pick that one even if the variable and switch are still true.

4

Event Conditions

There are six possible conditions you can manage per event page.

Switches: These are Boolean values, so either they are on or off. We can have two per event page.

Variable: An integer value. These are triggered when a value is greater than a specified value that you set. This will obviously be overwritten if you change it to a higher value and there is a separate event page to go along with it.

Self Switch: These are switches that only affect the event it is being created on. There are four possible states it can be in, labelled A, B, C and D.

Item: Checks if the player has a specified item in their inventory. If true, the event plays.

Actor: Checks if the player has a specified actor in their party. If true, the event plays.

5

Event Image

The image that will be used to represent the event in the game space.

6

Event Movement

Event movement refers to the movement of the event when it is not active. It can be fixed (stationary), Random (a random path), Approach (it will approach the player), and Custom (a unique route designed by you).

The speed is determines how fast the event will move from space to space, the frequency determines the amount of time between each movement.

This is also how you do idle animations.

7

Event Options

These options determine different elements about the event

Walking: This means that when the event is moving, it will play it’s walking animation (if it has one)

Stepping: This means that the character will do their walking animation in place.

Direction Fix: This means that no matter what, the character will not change directions.

Through: This means that if the event is moving, it will go overtop other events. If this is not checked off

8

Event Priority

Event priority refers to the layer that the event will be on. Visually, events will always appear overtop everything else, but whether or not they are above, below or on the same level of the players, depends on this option.

The trigger refers to what the players have to do to initiate the event. Action button means the player must walk up to it and press the action button. Player touch means the play must touch the event. Event touch is more or less the same thing. Autorun means it will run automatically when you enter the map (make sure it ends as this can cause an endless loop). Parallel is similar but it doesn’t stop the player to have the event play.

9

Event Contents

Event contents are the details of an event. There are many different functions you can have within the contents but we’ll go into detail elsewhere.
