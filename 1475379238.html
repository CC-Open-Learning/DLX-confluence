UI Notes:

Outro Modal UI:

The Outro Modal (OutroModal.uxml) UI panel is used for successful completions of all modes of seek and find (Beginner, Intermediate, and Advanced). For seek and find beginner, and intermediate, a simple success modal is displayed. For seek and find advanced, a summary of selected equipment is shown.

Beginner/Intermediate:

Advanced:

  • Uses OutroListEntry.uxml to populate and contain the list of items in Advanced mode

Summary UI:

The Summary (Summary.uxml) UI panel is only used for failures in seek and find Advanced Mode. This shows the correct, incorrect, and missed items for the selected scenario.

Advanced:

  • Uses SummaryEntry.uxml to populate and contain the list of items in Advanced mode

Compartment Workflow Notes:

Due to requirements changing late into the project, special conditions had to be placed throughout the system for the medication kit. This is especially prevalent in the compartment workflow because it was designed to have 2 layers of compartments instead of 4 (ALS bag → ALS bag top pocket → medication kit → medication kit top/bottom pockets). All of these sections have been highlighted so they can be easily found. The project contains many comments that go into more depth about the specifics of the medication kit.

Opening:

ExplorationManager:

Opening a compartment starts in MouseClickOnInteractable(). When this method determines that the learner has clicked on a compartment it starts the opening workflow for that compartment. The ExplorationManager maintains a currentlyExploringSubCompartment and currentlyExploringMainCompartment value to track the open compartments.

  • MouseClickOnInteractable() → Sub Compartment

    1. Exit the method immediately if another sub compartment is open.

    2. Exit the method immediately if we’re trying to open the medication kit pockets while the medication kit is closing.

    3. Set the currentlyExploringSubCompartment value and call OpenCompartment().

  • MouseClickOnInteractable() → Main Compartment

    1. Manually set currentlyExploringSubCompartment and currentlyExploringMainCompartment values if we’re opening the medication kit.

    2. Exit the method immediately if a sub compartment or main compartment is already open.

    3. Set the currentlyExploringMainCompartment value and call OpenCompartment().

  • Open Compartment()

    1. Call HandleFocusCompartment(), which disables interactivity of explorables outside of the compartment and makes a list of the disabled explorables.

    2. Start the HandleClickInteraction() coroutine on the CompartmentAnimBehaviour for the compartment, which triggers the opening animation.

    3. Invoke the FocusTaskCompartment event on self, which triggers TaskUISystem.HandleCompartmentClick(). This method focuses the explore mode checklist on the compartment.

    4. Call HideOutline() and HideTooltip(), which hide the respective element for the compartment.

    5. Call HandleDisableNavigation(), which prevents the user from using the walk points while in a compartment.

Closing:

ExplorationManager:

HandleCloseCompartment() is invoked when the leave compartment button is clicked on.

  • HandleCloseCompartment() → Sub Compartment

    1. Call HandleUnfocusCompartment(), which uses the list from HandleFocusCompartment() to enable interactivity of explorables outside of the compartment.

    2. Start the HandleClickInteraction() coroutine on the CompartmentAnimBehaviour for the compartment, which triggers the closing animation.

    3. If we’re closing the ALS bag’s top pocket, disable its intractability directly to prevent the medication kit from enabling it while closing.

    4. Call HandleCloseSubCompartment().

  • HandleCloseCompartment() → Main Compartment

    1. If we’re closing the medication kit, call HandleUnfocusMedicationKit(), start CloseMedicationBag() and exit the method early. [CloseMedicationBag()]

      • HandleUnfocusMedicationKit() exclusively enables the interactivity of explorables contained in the ALS bag’s top pocket.

    2. Call HandleUnfocusCompartment(), which uses the list from HandleFocusCompartment() to enable interactivity of explorables outside of the compartment.

    3. Start the HandleClickInteraction() coroutine on the CompartmentAnimBehaviour for the compartment, which triggers the closing animation.

    4. Call HandleCloseMainCompartment().

  • HandleCloseSubCompartment() → Contained Sub Compartment/Return to Main Compartment

    1. Reset currentlyExploringSubCompartment value.

    2. Call HandleFocusCompartment() on the main compartment, which disables interactivity of explorables outside of the main compartment.

    3. If the scene isn’t resetting, invokes the OnCloseSubCompartment event which triggers LeaveCompartmentButton.HandleShowButton(). This method displays the leave compartment button so the player can exit from the main compartment.

    4. Invoke the FocusTaskCompartment event on the main compartment, which triggers TaskUISystem.HandleCompartmentClick(). This focuses the explore mode checklist on the main compartment.

  • HandleCloseSubCompartment() → Standalone Sub Compartment/Mimics Main Compartment Workflow

    1. If the scene isn’t resetting, invoke the OnCameraReset event and the OnCloseCompartment event.

      • OnCameraReset triggers PlayerHandler.ResetPlayerCameraRotation, which recenters the camera.

      • OnCloseCompartment triggers TaskUISystem.HandleCloseCompartment, which resets the explore mode checklist and LeaveCompartmentButton.HandleResetButton, which disables and hides the leave compartment button.

    2. Call HandleEnableNavigation(), which allows the user to interact with the walk points.

  • HandleCloseMainCompartment()

    1. Reset currentlyExploringMainCompartment value.

    2. If the scene isn’t resetting, invoke the OnCameraReset event and the OnCloseCompartment event.

      • OnCameraReset triggers PlayerHandler.ResetPlayerCameraRotation, which recenters the camera.

      • OnCloseCompartment triggers TaskUISystem.HandleCloseCompartment, which resets the explore mode checklist and LeaveCompartmentButton.HandleResetButton, which disables and hides the leave compartment button.

    3. Call HandleEnableNavigation(), which allows the user to interact with the walk points.

  • CloseMedicationBag() → Not Resetting

    1. Call CameraManager.UpdateCamerasBeforeMedKitClose(), which manually sets the previousCamera to the ALS bag top pocket.

    2. Start the AnimateCompartment() coroutine on the medication kit CompartmentAnimBehaviour and wait for it to finish. This coroutine triggers the close animation and exits when it’s complete.

    3. Invoke the OnExitMedicationBag event, which triggers LeaveCompartmentButton.HandleInitalizeButton(). This method enables and displays the leave compartment button.

    4. Call CameraManager.UpdateCamerasAfterMedKitClose(), which manually sets the CameraManager values to the correct values for the ALS bag top pocket.

    5. Invoke the FocusTaskCompartment event on the ALS bag top pocket, which triggers TaskUISystem.HandleCompartmentClick(). This focuses the explore mode checklist to the ALS bag top pocket.

    6. Manually set the currentlyExploringSubCompartment and currentlyExploringMainCompartment variables to the correct values for the ALS bag top pocket.

  • CloseMedicationBag() → Resetting

    1. Start the ResetCompartment() coroutine on the medication kit CompartmentAnimBehaviour, which makes it instantly close.

    2. Invoke the FocusTaskCompartment event on the ALS bag top pocket, which triggers TaskUISystem.HandleCompartmentClick(). This focuses the explore mode checklist to the ALS bag top pocket.

    3. Manually set the currentlyExploringSubCompartment and currentlyExploringMainCompartment variables to the correct values for the ALS bag top pocket.

Resetting:

ExplorationManager:

Resetting a compartment follows a similar workflow to closing it, with a few differences. HandleResetCompartments() is triggered from various events whenever the learner returns to the menu or restarts a scenario.

  • HandleResetCompartments()

    1. Start the ResetCompartments() coroutine.

  • ResetCompartments()

    1. Call CameraManager.ToggleCameraReset(true), which makes camera transitions cut immediately.

    2. If there’s an active sub compartment, start the ResetCompartment() coroutine on its CompartmentAnimBehaviour and wait until it’s finished. After waiting, call HandleCloseSubCompartment().

    3. If there’s an active main compartment, check if it’s the medication kit.

      • If it is, start the CloseMedicationBag() coroutine and wait until it’s finished. [CloseMedicationBag()]

      • If it isn’t, start the ResetCompartment() coroutine using the main compartment's CompartmentAnimBehaviour and wait until it’s finished. After waiting, call HandleCloseMainCompartment().

    4. Check if either currentlyExploringSubCompartment or currentlyExploringMainCompartment are null. If they aren’t, we just reset the medication kit and still need to reset the ALS bag top pocket.

      • If they’re not null, call the ResetCompartments() coroutine again. [ResetCompartments()]

      • If they’re null, call CameraManager.ToggleCameraReset(false), which returns camera transitions to the default mode.

    5. Call HandleUnfocusCompartment(), which uses the list from HandleFocusCompartment() to enable interactivity of explorables outside of the compartment.

    6. Invoke the OnSceneResetEnd event, which triggers SeekAndFindManager.SetAnimationSpeed(). This method resets the AnimationSpeed to the appropriate value based on the current mode.

General:

CompartmentAnimBehaviour:

This script is required as a component for all compartments and handles all of the animation logic for the compartment. It uses a bool (IsOpen) to track the current state of the compartment. AnimationSpeed is a public static variable that determines the speed of all compartment animations.

  • HandleClickInteraction()

    1. Calls ChangeCameras(), which activates the compartment camera when opening or the previous camera when closing.

    2. Starts the AnimateCompartment() coroutine.

  • AnimateCompartment()

    1. Call ToggleComponentsBeforeAnimation(), which toggles the intractability of relevant interactables depending on whether the compartment is opening or closing and hides the leave compartment button by calling LeaveCompartmentButton.HandleHideButton().

    2. Swap IsOpen value to reflect the new state.

    3. Start the HandleCompartmentAnimator() coroutine once for each animation (compartmentAnimator, auixiletyAnimator, explosionAnimator). The animations triggered will be based on the specific compartment and whether it’s opening or closing. This coroutine will not exit until all animations are finished.

      • When opening, the order is compartmentAnimator (wait until animation finishes) → auxileryAnimator → explosionAnimator (wait until animation finishes).

      • When closing, the order is explosionAnimator (wait until animation finishes) → auxileryAnimator → compartmentAnimator (wait until animation finishes).

      • The auxileryAnimator is used for “fluff” animations and doesn’t exist for most compartments. It was originally implemented to facilitate the medication kit.

    4. Call ToggleComponentsAfterAnimation(), which toggles the intractability of relevant interactables depending on whether the compartment just opened or closed.

      • When open, call LeaveCompartmentButton.HandleInitalizeButton(), which enables and displays the button.

      • When closed, determine if the compartment should enable itself by calling ShouldEnableCompartment(). This is mainly used to prevent sub compartments from enabling themselves if their animation finishes after the main compartment.

  • ResetCompartment()

    1. Exit immediately if the compartment isn’t open.

    2. Wait until the compartment isn’t animating.

    3. Set AnimationSpeed to AnimationSpeedResetting and start the AnimateCompartment() coroutine.

    4. Wait until the compartment is done animating before exiting.
