VARLab : Workflow for 2D Inspection

Created by Amila Abeygunasekara, last modified on Dec 15, 2022

Terminology

Below is an overview of the classes and objects used in the 2D inspection of the LSM6.02 Heavy Equipment - Bulldozer digital learning simulation.

Classes

Class

Description

Zoned2DInspection

(Derived from SpecializedInspectionElement)

Zoned2DInspection is used as the starting point of the 2D inspection. It is a component of an element of the bulldozer 3D model.

Instances that have Zoned2DInspection attached can be included in the Inspection Order of the Inspection Manifest like other regular inspections.

Zoned2DInspection is considered as a 3D inspection because it is a part of the bulldozer 3D model.

Clickable2DInspection

(Derived from MultipleChoiceInspection)

Clickable2DInspection is a component of each rectangle zone on the 2D inspection window. This class is responsible for handling mouse clicks and poking InspectionStateMachine.

Clickable2DInspection type of instances are excluded from the Inspection Manifest.

Zoned2DInspectionUI

(Derived from UserInterface)

Zoned2DInspectionUI is a component of the 2D Inspections GameObject, a child of the Window Manager GameObject. It controls the 2D inspection window (User interface).

Zoned2DInspectionUI's lifecycle is controlled by the 2D Inspection Workflow of the Bulldozer State Machine.

InspectionCollection

This class acts as an identifier for the container of Clickable2DInspection GameObjects.

Additionally, it ensures proper serialization of Zoned2DInspection's Inspection Container field by ensuring the object type.

Creating a 2D inspection

Step 1: Creating a 2D inspection container instance

Go into the prefab editing mode of the 2D Inspections prefab by clicking on the rightmost arrow.

Create an empty GameObject inside 2D Inspections and give it a name. Make sure that the new GameObject is below the Floating Background GameObject. For this example, letâ€™s consider the Engine Inspection GameObject shown below.

Go to Unity inspector and insert Image and Inspection Container components into the newly created GameObject. The Image is used as the background of the 2D inspection.

Step 2: Creating inspection elements

Create a new GameObject as a child of the Engine Compartment and give it a name.

Add the following components and serialize the required fields appropriately.

Note: Since this inspection is 2D, the Camera field must left empty.

Finally, resize and place the inspection element as you like.

You now have successfully created the first inspection element. You can duplicate this inspection element if you need to insert more elements.

After duplicating, the only changes that needed to be done are:

  • Update the name of the new inspection element

  • Update the Clickable 2D Inspection component

  • Update the Simple Tooltip component

  • Size and placement of the new inspection element

After creating all the required inspection elements, it is safe to exit from the prefab edit mode.

Step 3: Adding the 2D inspection entry point into the bulldozer 3D model

Select the 3D mesh object of the bulldozer that you want to invoke the 2D inspection.

Add the following components to the above GameObject.

Serialize the 2D inspection container instance created in Step 1: Creating a 2D inspection container instance into the Inspection Container field of the Zoned 2D Inspection component.

Step 4: Inserting the inspection into the Inspection Manifest

As the final step, click on the Bulldozer prefab and navigate to the Inspection Manifest component.

Click on Find Inspection Elements to insert the newly created 2D Inspection into the Inspection Order list.

Place it in the correct order and override changes to the Bulldozer prefab.

Congratulations! The 2D inspection you created is now ready to use. You can now go ahead and test it.
