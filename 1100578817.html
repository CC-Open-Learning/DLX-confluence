VARLab : Climate Change V3_Network Testing (Web)

Created by Sana Javeed, last modified by Cristina Barillas on May 14, 2024

This process guide should be followed when assessing a DLX simulation build as a potential playable or releasable candidate build. This would include receiving incremental or ‘hotfix’ builds from an external vendor.

The goal of this network testing protocol is to arrive at a level of confidence that the sim can be played in its network context with little to no worry of negative impact on CE’s, students, or other testers.

TEST BUILD: Climate Change Scorm_534_v3.0_Release_Build 28.Mar.24 - VARLAB-DEV1 (desire2learn.com)

MODE: MULTIPLAYER

Overview

  1. Baseline Network Systems Tests – Track a minimal, but necessary set of network performance metrics

  2. Multiplayer Experience Tests – Similar to above, but extends testing for the multiplayer context 

Who does this testing?

Anyone with familiarity with computer networks and some of the tools mentioned below that would assist in capturing quantitative measures of network performance. IT/Ops personnel are a good standard of the experience required. Anyone may consider teaming up to conduct these tests (esp. when Multiplayer is in context for the testing).

IT/OPS need to ensure they test both for A) Baseline Network Systems Tests and collaborate with others to test for B )Multiplayer experience tests

A)     Baseline Network Systems Tests

To document baseline performance, use tools such as Task Manager and/or Wireshark running on the user’s PC AND from the web host/server to collect following metrics:

  1. Latency: Measure the time it takes for data to travel between players and the server.

  2. Bandwidth: Monitor data transfer rates.

  3. Packet loss: Detect dropped packets.

  4. Server load: Assess how well the server handles multiple connections.

  5. Test under various conditions (e.g., different browsers, network speeds) to identify bottlenecks and optimize performance.

B)    Multiplayer Experience (MP) Tests

  1. Record wait time(s) to load and launch the sim, including lobby access (if applicable). How long until you can play with others, assuming reasonable conditions for all?

  2. Is progress saved automatically or manually by the player? (if a requirement)

    1. Is this achievable according to spec in the MP context?

  3. What is the outcome if the player closes their web browser?

    1. What happens to the rest of the game experience for the other players?

    2. Can the MP session be easily rejoined by the disconnected player?

  4. Disconnect from the network or go into airplane mode. What happens when the network goes away?

    1. What happens to the rest of the game experience for the other players?

    2. Can the MP session be easily rejoined by the disconnected player?

  5. Throttle bandwidth lower (some browsers support this) and/or access the sim from a known slow/poorly performing network. Document performance impact (e.g. appx what bandwidth or performance impact was inflicted?).

    1. What happens to the rest of the game experience for the other players?
