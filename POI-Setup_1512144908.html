VARLab : POI Setup

Created by User bf53c, last modified on Jan 15, 2025

Definitions:

  • POI: Camera view used to get a closer look at an area. Stans for “Point Of Interest”.

  • Main POI: Highest level POIs that have no parent POIs.

Creating a POI:

  1. Add the POI, Interactable, Collider and Outlinable components to the GameObject for the POI.

  2. If the POI needs a “Box” outline, follow the steps here. If not, double-check that the Outlinable component is registering the correct meshes. The POI GameObject doesn’t have to contain the meshes for your outline, but they’ll have to be somewhere in its child GameObjects.

All POIs will eventually use dynamic outlines (see related ticket)

  1. Add the GameObject to the EventSystem interactables list.

  2. Place the GameObject appropriately in the hierarchy so it’s the child object of the POI where it’s opened. If the POI isn’t meant to have a parent, place it at the highest level and check the “IsMainPOI” box on the POI component.

  3. Disable the Interactable and Collider components if the POI isn’t a MainPOI.

  4. Set the “MouseClick” event on the Interactable component to this GameObjects POI.open() method.

  5. Change the “poiName” value to the name of the POI.

    1. If the name isn’t on the list, add it to the POIName enum, and it should show up.

  6. Populate the childPOI and explorables lists with the appropriate GameObjects. One or both of these lists can be left empty if needed. Explorables don’t need to be children of their POIs, but child POIs must be children of their parents.

  1. Create a CinemachineVirualCamera for the POI, as seen here.

Creating The POI Camera:

  1. Enter the “Camera” prefab and create a new CinemachineVirtualCamera object.

  2. Adjust the position/rotation of the camera so it’s in the desired location (make sure to leave room for the Progress Indicator on the right side of the screen).

    1. The camera view can be seen in scene view via the camera window, but it’s often easier to modify values during play mode.

  3. Set the priority of the CinemachineVirtualCamera to 0 and rename the camera to match the intended POI.

  4. Disable the CinemachineVirtualCamera GameObject.

  5. Reference the camera in the poiCamera field of the POI (must be done at the Main Scene level).
