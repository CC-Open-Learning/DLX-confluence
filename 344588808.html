VTS Avatar Deployment

Avatars in VTS are created by Serious Factory (SF) and there are several Avatars to choose from. These Avatars include a variety of appearances, attire, and level of animation, but are not further customizable to meet specific project needs.

According to SF, Avatars have separate head and body components that are combined to create a specific character. Therefore, one head asset, including expressions can be attached to a different body to create a unique character.

For example, SF used the Zaifeng head asset for two characters - Zaifeng Striped Shirt and Zaifeng Doctor.

NOTE: The current version of the SF Avatars is 3rd generation. Earlier generation Avatars are less expressive as they do not include the VTS animation module. To make your Avatar as lifelike as possible, select 3rd generation Avatars as your foundation.

Serious Factory Custom Avatar Options

There are two ways to create a custom Avatar. Note that both methods require SF involvement.

  1. Combine Existing Avatar Assets

  2. Modify Existing Avatar Assets

Combine Existing Avatar Assets (recommended)

If the assets you require exist in other 3rd generation VTS Avatar, identify the assets to be combined, and request SF create and implement the new Avatar in VTS.

Example: Combine a young face with a pleasant expression with a casually dressed body.

Modify Existing Avatar Assets

If the assets do not exist in SF, we can alter existing Avatar assets using 3D modeling or texturing software. This requires SF to provide source files for Avatar assets and their texture files, and SF recommends that hands and faces are not altered so the animation is not impacted. This limits us to altering clothing and other assets, and we may be required to do the rigging and skinning of assets again.

Both methods require SF to implement the altered Avatar in VTS. Once uploaded on the VTS server, our custom character is available to us. These custom characters won't be available to anyone else, and they will automatically be available to us after we refresh our VTS editor or restart the software.

Required skills for Avatar creation (fully rigged and animated)

Avatar creation requires good knowledge of anatomy and silhouette. If these things can be achieved then a 3D model can be created and textured. Clothes and other assets can be easily made to help dress our character well. These characters will require their heads and bodies made separately. We can talk in more detail regarding both facial and body requirements,

FACE: The face requires different blend shapes to be created to display various expressions and actions that can be done only with the face. These blend shapes will then help form the emotion wheel… there will be normal blend shapes that encompass the vowels A,E,I,O,U … and many other expressions that we use to mix and match to create an animation for emotions.

BODY: The body has to be properly rigged and skinned so that we can create proper animations on them. These animations also can be made in synchronization with emotions. Some animations can be standalone and have a facial expression associated with it (just like VTS has Animation Block).

Softwares that we require for task like this might include:

Modeling/ Animation - Autodesk Maya/ 3Ds Max, Blender, etc.

Sculpting - Zbrush, Mudbox, Sculptris, etc.

Retopology - Same as modeling , 3D Coat, zbrush etc.

Texturing - Photoshop, Substance painter

Recommendations for Avatar Creator Development

Avatar creation in VTS must be carefully considered. It is time and resource-intensive, with long-term impacts on the appearance of the simulation. Other simulation development software may provide more options to create unique characters with fewer resource demands.

At a minimum, Avatar Creation should allow the combination of the following assets to create a unique character:

  • Head

  • Body

  • Other assets that go on top (eg: hats, glasses, shoes, etc.)

Custom Avatar Creation should be considered as a potential feature in the Rapid Development or Core Engine projects, as it has the potential to improve the quality of the simulations we create. Game environments provide many examples of character customization, in which the user has the capability to customize many features of their character.

Buying Avatar VS creating them from scratch

Are there any sources for buying avatars that could be listed here? Are there any known limitations to importing purchased assets into our existing tools? Are there any considerations for choosing new tools/building our own?

Buying is a viable option available to us instead of creating an asset from scratch. This decision has its own pros and cons which need to be weighed and an optimal solution must be sorted out. Here we strike between time and resource value. Buying assets should be done when we know we can save quite a bit of time and resources which will help the company.

These assets need not be a complete asset, it can be a starting point for some artists to modify the existing bought asset and work towards modifying it to suit our requirements. We need to know how much time will we save in buying an asset like that and how we can work on the asset to make it even better.

Creating an asset from scratch requires a lot of research and skills for character anatomy. It is easier when a base model is found and it can then be tweaked to get the desired look. The texture of these assets can also be changed while making the edits.
