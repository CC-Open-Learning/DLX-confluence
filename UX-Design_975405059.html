User ( Persona)

 

 

Who

A first- year student in the Electrical program at the School of Trades and Apprenticeship.

They have no previous field experience.

They are expected to practice on this simulation after class hours like a PRE-Lab activity before the hands-on workshop.

They need to know how to read diagrams, look up codes, plan a project, as well as install and test the circuit.

Goals

To learn and practice how to install and test a basic circuit in an unfinished wall of a residential and industrial building.

To get into them into the habit of using the code book rather than memorizing it.

To get used to the jargon being used in the field

User Behaviors

  • Does not prefer to bring laptop to classroom as the program has more tactile hands-on tactile workshops

  • When needed, they use the kiosk machines or borrow a laptop at the Reuters campus library

Minimum System Requirements

  • 13" screen or greater (recommended resolution 1920x1080)

Challenges

  • Students are not well-prepared before the lab class.

  • After a project submission, students learn about the results after the graded evaluation.

Due to time constraints in the lab class, the instructor would like to make best- use of the time spent during the hands-on workshop by having students come prepared beforehand.

HMW (How Might We…)

Enable students to have practiced outside class hours, while giving them feedback on the work completed before the hands-on workshop.

  • The students have no experience in the field

  • The student needs to learn the jargon

Making changes AFTER Feedback

Information from the Content Expert

Q1: Should we limit the student to ONLY correct their mistakes? Chris: YES.

Q2: Should we reduce the number of options available when fixing a mistake? Chris: YES - After feedback, reduce the number of choices down to 2 options (one correct, one incorrect).  If still incorrect, only the correct option is available on round 3.

Q3: Should we limit the number of times a student can ask for feedback?  Chris: YES - Let students take responsibility for their work, they can only ask for feedback after they have revisited all "error" tasks in the feedback.  They cannot address one error and then get feedback again. The Feedback button will remain disabled until all 4 tasks are once again "completed"

Learner can make changes to the errors made in the environment.

At every round of feedback ( eg: making changes and asking for feedback again) the number of possible options to place the boxes in the environment reduces.

For eg: After the 2nd time asking for feedback, choices in location options are 2 ( one correct, one incorrect ).

After the 3rd time asking for feedback, only one correct choice exists.

Task-based workflow

In the previous design, options were available based on the component ( eg box) selected. The issues were the various unrealistic tracking if the student makes wrong installations.

Options provided will be task based.

eg. Select and mount a fan box (irrespective of the type of component) will show the options to install in the environment.

To DISCUSS: As the number of options provided at the time of installing could be enormous ( for boxes and cables), limiting the options would be easy to track. Furthermore, early feedback at every task would prevent multiple lines of errors for the student. Hence suggestion to CE

  1. To limit the number of boxes to install and

  2. To provide early feedback from the red seal ( and not during rough-in inspection)
