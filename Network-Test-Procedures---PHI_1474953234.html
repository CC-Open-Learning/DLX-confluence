VARLab : Network Test Procedures - PHI

Created by Carolina Naoum Junqueira, last modified on Nov 21, 2024

Overview

These baseline tests intend to document network and web browser performance of the Public Health Inspection - Restaurant DLX as a WebGL module in eConestoga.

These test procedures are being performed on the PHI-24F5.4 - 12NOV2024 build which is proposed as release candidate. The file size is 208MB (compressed).

Baseline Network Systems Tests

These baseline network tests aim to track a minimal but necessary set of network performance metrics.

Bandwidth

  • This was tested by checking the network speed using Speedtest by Ookla.

  • We timed the load time 5 times for each device and took the average.

  • Started the timer when refresh/build link was clicked and stopped the timer when the welcome screen was loaded.

Server Load

The web host for all modules is eConestoga so no measurement can be performed server-side. Loading the simulation itself is dependent on eConestoga servers and the Unity WebGL runtime.

Latency

Using Wireshark we checked the response time for Brightspace, Playfab, and CloudSave.

Made a recording of the DLX load process using Wireshark and reviewed all requests and responses made to these services. These are the lowest and highest response time for each service

  • Brightspace: 0.002152 - 0.002624 seconds

  • Playfab: 0.002839 - 0.012019 seconds, 1 request failed*.

  • CloudSave: 0.06326 - 0.072291 seconds

*Note: transient failures are normal and automatically retries if there is a failure. We validated the data in Playfab and all the expected logs were present.

Packet Loss

Because we are unable to capture the packet loss we are tracking retransmission request. The average retransmission request is 0.3%.

Single-Player Experience

Single-Player Experience Tests intend to validate the stability and support of single-player experiences with the sim.

Load Time

The load time changes according to network speed ranging from ~40 secs to ~2mins.

Fullscreen

The DLX is intended to be launched as embedded content within eConstoga tab, and the screen will resize to fill the available space.

Learners can enter and leave fullscreen mode with no issues.

Saving Progress

The DLX saves student progress two different ways: inspection records (Inspection Log, Gallery, Activity Log) and player location.

  • Inspection records are saved to Azure every time the player clicks on an item, takes a photo or temperature reading, or completes an inspection.

  • Player location is saved every time the player enters a new area, such as the dining room or kitchen. When the player chooses to continue a saved inspection, they will be returned to a default position in the area they last entered. Even if the player was at a different position in the area when they left, they will start from the default position.

Playfab total time played is triggered in 180 second intervals, this interval increase to ~240 seconds when the DLX is not in focus. Total time played event is also triggered when the learner clicks on end inspection.

Offline

Loss of network connection does not interrupt the learner’s ability to play the DLX.

  • Loses internet while playing and does not regain access before browser is closed: inspections done while offline will be lost. When the learner selects continue the next time they play those inspections will be missing.

  • Loses internet while playing and regains access before closing the bowser: inspections done while offline will be saved. Playfab events will also get triggered.

  • Player can not load the DLX when they are offline.

Bandwidth

Lower bandwidth increases overall load times but has no effect on the simulation once loaded, as there are no dependencies on external network resources after the initial load.

 

Multiplayer Experience Tests

Public Health Inspection has no multiplayer component, so no tests of this kind are being performed.
