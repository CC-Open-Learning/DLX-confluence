Before Beginning

This document makes some assumptions, to begin this assumes the art assets have already been imported to the engine, and that the materials have been applied to the models which have then been prefabbed. Additionally it assumes you have a basic working knowledge of the Prefab system in unity.

If you need to import the assets please see this page: https://varlab-dev.atlassian.net/wiki/pages/resumedraft.action?draftId=1070006323&draftShareId=c7cab80c-7ccf-45bb-bd5b-39f0a55e734c

To learn about prefabs please see: https://learn.unity.com/tutorial/prefabs-e#5f49e5bcedbc2a280d7cf66f

Creating A New Inspectable

Step 1: Adding The Premade Empty Inspectable

Within the Assets/Prefabs/Scene folder there is a prefab called InspectableObject.prefab

You will need to simply drag this into the hierarchy in the left side of the editor.

Now the last step will be to right click the “InspectableObject” and unpack the prefab as shown below

Additionally you can rename “InspectableObject” to whatever the new inspectable should be called.

Step 2: Placing The Asset

The next step were going to start working with the asset you are placing, for this example I will be placing a box on a table.

Firstly find the asset within the Assets/Prefabs/Inspectables. (This is where it should have been placed after importing) if you can not find it, use the search bar.

Your looking for a prefab like this

Do not use an asset that looks like this

When you have found the correct asset drag it into the scene window to give it an initial placement.

now we want to place it within the game world where we want it.

If your inspectable has multiple visual assets (dented can and normal can) you can repeat the above process for each asset but can paste the transform to quickly place the other objects in the same spot.

Step 3: Combing Our Objects

With steps 1 and 2 done we now have

  1. an empty “inspectable object”

  2. our asset/assets

Notice how the Box_Styrofoam_Medium is still a Prefab! This is important DO NOT unpack this one!

The next step is to combine these into a single object that we will then create a Prefab from.

First take the transform of the asset object we placed by copying the component.

Paste that into the “MyNewInspectable” (what ever you named your empty inspectable).

Now our locations should match for both objects! Now we want to fix the hierarchy. To do this we will simply drag the object under the empty compliant object so that it now looks like this.

For objects that have multiple states if the visual asset is the same just duplicate(ctrl + d with the object selected) the original object and drag it onto the “Non compliant” for states with different visual assets (normal can vs dented can) you can drag the dented one under non compliant as an example.

Examples

Examples Click me to expand!

Next we want to create a new Prefab of the combined objects, to do this you will simply click “MyNewInspectable” (which you renamed) from the hierarchy and drag this into the according location within the folder structure. This will likely be something like Assets/InspectableObject/Kitchen the under lined word replaced with the location from your ticket.

Step 4: Adding Our Object To The Location

Now we have the following

  1. Our Prefabbed game object with the assets added

  2. Our assets placed where we want them

Our next step is to add them to the correct location within the hierarchy! For this example I have placed this box in the Back Kitchen so I will need to find that location in the hierarchy under Inspectables and POIs followed by Areas, find the area that matches where you are placing the object and expand it, then expand Inspectables, you can then drag your “MyNewInspectable” within the hierarchy onto Inspectables. when finished it should look like the following image.

Great! But what does this green plus mean? Our locations are themselves Prefabbed as a collection containing our Inspectable Prefabs, the green plus means we added an object in the scene but it hasn’t been added to the parent Prefab. To fix this we simply right click our added object and add the object to the parent prefab as shown below.

Once that is clicked the green plus should disappear and now we have our object prefab added to our location prefab.

Step 5: Setting Up The Components

Still following along? Great! Lets recap again, we should have the following

  1. A Prefab of “MyNewInspectable” with the asset(Styrofoam_Box) prefab added under it’s compliant/non-compliant states

  2. The above Prefab added to the according location Prefab

With all that done our hierarchy, and physical location of the objects within the scene should all be finished so now we need to setup the details about our object! Our Inspectables have 3 major components that are important to setup which we will now go over.

But first we want to make sure we are working in the correct location! We most likely will want to work within our “MyNewInspectable” Prefab so to do this we can double click our prefab in Assets/InspectableObject/BackKitchen OR Click this arrow

When done your screen should something like this.

Now to begin setting up components!

Inspectable Object Component

Several of the fields to fill out are rather straight forward, the ticket will supply the name and location which can be filled out, the location will be selected from a drop down list and you will type the name in.

Scenario states is a little trickier, you will click the + to match how ever many states the object has (ie. compliant + non compliant would mean 2) and it should look like this.

Now we want to drag the game objects “InspectableObjectCompliant” and “InspectableObjectNonCompliant” to the game object references.

Now we will need to create the information that is associated with the game objects so navigate through the folder structure to Assets/Scriptable Objects/Inspectable Object States/*your location*/*Inspectable Name* and you will right click Create → Scriptable Object → Inspectable Object and create as many states as specified in the ticket. In my example I will be creating 2, rename them to match your objects name.

Now we fill in the information that is on the ticket.

With these made and filled out we then will go back to our Inspectable Object Component and match the states to the game object references.

Edge-case: Toggleable
How to add a Toggleable

This is an edge case where certain inspectables may have a portion that is “toggleable” such as a lid that needs to be removed when inspected or an object that needs to move to be inspected. Here is an example of the “Inspectable Object” component from an existing inspectable in the scene (Large_Meat_Box)

When the object is clicked on to be inspected the game objects that in the toggleables list will toggle from on->off or off->on. The “should toggle” box needs to be checked to enable the toggling. this allows a box in the scene to be covered, but then when clicked to be inspected in the picture within the inspection window the lid will be removed.

Outlinable Component

This is the component we will be setting, we still want to do this in the Prefab editor mode!

This component is easy to setup! we will click the plus then select “Add all basic”

lastly confirm that the outlinable component is unchecked

This component is now finished!

Box Collider Component

This component is about setting up the box colliders for the various states in this example we will set a box collider for each state object (compliant and non compliant) however as you get more comfortable with setting up inspectables the collider can actually be shared between both by placing it on the parent. Again ensure you are still in the Prefab editor mode!

We will select InspectableObjectCompliant and see this in the inspector

This will allow us to edit the box collider in the scene view

If there is a mis-match between the transforms of the parent, the state object, and the model it will become apparent now. It is relatively easy to fix, and at the end you want everything to basically be zero’d with highest level parent.

You can click and drag each face until it about matches the size of the object. when done right click the component and copy it to the box collider on the “InspectableObjectNonCompliant”

This will ensure that the components are the same. lastly ensure that the Tag value is set to InspectableObject on the game objects that have the colliders.

And now our colliders are set!

Step 6: Final Touches In The Scene

We are almost done! Now we want to exit the prefab editor by hitting this arrow.

This is everything for the inspectable completed, now we have to update our Inspectable Manager. In the hierarchy click Master Components and select the Inspectable Manager. We then need to click on the “Generate ID’s” button.

This concludes all the work in the scene!

Setting Up Scenario Files

If the inspectable being added has only One State, then the scenario files will automatically be set up when the inspectable has been given a state and the inspectable manager has been updated.

With our inspectable now completed we need to set up the 6 scenario files so each scenario knows which state of our object to use. To open the menu that sets our scenarios along the top menu bar and select the PHI tab and click Inspectable State Editor.

This will bring up a new window, now in the top select the drop down besides “Load File” and select scenario 1. This will load the file and now you need to find the empty option in the list as shown below.

Now click the drop down besides our inspectable and set the state to what the ticket says it should be for Scenario 1. Once this has been selected click “Update Scenario File” in the bottom of the window.

Now repeat the above steps for scenarios 2-6 matching the scenario to the state specified on the Jira ticket updating each file as you go. When you are done this everything should be okay to test! Run the game as normal and try highlighting and inspecting your added object and if everything runs well you should be done!
