VARLab : CCSIF Trucking’s use of Unity’s Timeline package

Created by User 763fb on Dec 12, 2024

CCSIF Trucking includes two cutscenes that involve several animations playing at different times. To manage these animations and their timing, we’re using Unity’s Timeline package, a tool designed to handle sequences and interactions without extensive scripting.

This documentation provides an overview of the Timeline package features that will be used in CCSIF’s cutscenes. By following this guide, you’ll learn how to set up and use the Timeline to organize and execute the necessary animations, audio, and events effectively.

How to Download the Package

To download the Timeline package, follow these steps:

  1. Navigate to Window > Package Manager.

  2. Set the filter to Packages: Unity Registry.

  3. Search for the Timeline package and click Install.

Windows and Components

Once the package is installed, open the Timeline window:

  1. Navigate to Window > Sequencing > Timeline.
    This is where you will create and modify timelines.

Creating Your First Timeline

To create your first timeline:

  1. Create a new Empty GameObject in your scene.

  2. Open the Timeline window and click the Create button.

This action will open a file explorer window. Choose a location to save your new .playable Timeline asset. Once created, Unity automatically adds the Playable Director component to the selected GameObject and references your new Timeline asset in the Playable property field.

Getting Started with Timeline

Now that your setup is complete, you have the three essential components needed to start working with timelines:

  • A Timeline Asset.

  • A GameObject with the Playable Director component.

  • The Timeline Window, opened and locked to your selected GameObject.

The Timeline window features a similar layout and functionality to the Animation window. If you’re familiar with Animation tools in Unity, you’ll find much of this knowledge transferable when using Timeline.

Different Tracks to Add

The Timeline window allows you to add a variety of tracks to build your sequences. However, CCSIF Trucking will primarily use the following four track types:

  • Animation Track

  • Activation Track

  • Audio Track

  • Cinemachine Track

Animation Track

An Animation Track is used to control animations on objects in your scene. Here’s how to set one up:

  1. Add an Animation Track in the Timeline window.

  2. Assign a reference to an object with an Animator component. If the object doesn’t already have an Animator, Unity will prompt you to add one.

    • For this example, I’ll add animation tracks for a professionally designed box and some assets that I want to exit the box.

  1. Drag each object's animations into the timeline. Adjust the timing to control when the animations will start.

    • In this example:

      • The box open animation plays immediately after the timeline starts.

      • The objects’ exit animations are set to play just after the box finishes opening.

  1. To trigger the timeline, reference the PlayableDirector's Play() method through a Unity Event.

By doing this, we can create a synchronized set of animations triggered by a single event—all without writing any code. This entire setup can be completed in just a few minutes.

Audio Track

An Audio Track works similarly to an Animation Track but is used to control audio clips. Here's how to use it:

  1. Add an Audio Track in the Timeline window.

  2. Assign a reference to an Audio Source component. If the selected GameObject doesn’t have an Audio Source, Unity will prompt you to add one.

  3. Drag audio clips into the track and align them with your animations.

    • For example:

      • I aligned audio clips with the box opening and closing animations.

  1. Use the Volume Curve feature to adjust the volume for all clips on the track dynamically.

This approach allows you to sync audio with animations seamlessly, enhancing the overall experience.

Signals

Signals are a powerful tool for invoking events directly from your timeline. They consist of three key components:

  1. Signal Asset

  2. Signal Emitter

  3. Signal Receiver

Here’s how to set up and use Signals:

  1. Open the Timeline Window and enable Show Markers to make the marker track visible.

  1. Click on the marker track and select Add Signal Emitter. Place the emitter at the desired position on the timeline.

  2. In the Inspector, add a Signal Receiver.

    • This will automatically add a Signal Receiver component to your timeline’s GameObject.

  1. On the Signal Receiver, click Add Reaction.

    • Open the dropdown menu and select Create Signal to create a new Signal Asset.

  2. Assign the newly created Signal to both:

    • The Signal Emitter on the timeline.

    • The Reaction in the Signal Receiver component.

With this setup, you can trigger any number of events directly from the timeline. Signals provide a simple and easily adjustable way to integrate event-driven behavior into your sequences.
