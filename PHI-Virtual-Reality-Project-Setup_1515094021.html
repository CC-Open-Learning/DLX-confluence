VARLab : PHI Virtual Reality Project Setup

Created by User 55f04, last modified on Dec 12, 2024

We followed the instructions at CORE XR to set up the project for VR. We ensured that the project had the following packages:

  • OpenXR v1.10.0

  • XR Interaction Toolkit v2.5.4

See Jira issue PHI-1335 - As a developer I want to configure CORE XR for the PHI to start implementing VR capabilities. Done PHI-1335 - As a developer I want to configure CORE XR for the PHI to start implementing VR capabilities. Done for detailed implementation:

Once the setup, including the installation of the packages is complete as per the instructions in the link above. Search for XR system under the project and you will notice a prefab that belongs to the com.varlab.xr package.

You do not edit this or use this in the scene. You would want to drag it into the scene first and then create its prefab variant in a folder that encompasses any XR related files. This variant of the XRSystem prefab can be modified without changing the package file. For my implementation I have it under the Assets/Prefabs/ folder. See Jira issue PHI-1337 - As a developer, I want to add the XR system to the project to implement VR functionalities. Done .

The XRSystem prefab has everything that is needed to get the simulation running in VR. Note that I added the new variant of the prefab under master components object in the main scene (You can drag and drop it anywhere as long as it is in the main scene.

You also will need add the packages to the Assembly Definition References to ensure the Root Namespace (in this case VARLab.PublicHealth) is using them:

Make sure to link the Meta Quest 3 headset to your computer using the Meta Quest Link application. Launch the simulation while Link is active and you will find yourself within the simulation in VR.

There are generic controls that are active by default. Walking and Turning are the few that were noted as the default controls.
