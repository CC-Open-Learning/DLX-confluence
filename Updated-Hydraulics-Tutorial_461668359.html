VARLab : Updated Hydraulics Tutorial

Created by Duane Cressman (Unlicensed), last modified on Dec 01, 2021

This tutorial will rely heavily on the Graph Management system. Please read the Documentation if you are not familiar with the system. It can take a bit to figure out all the features, but once you get it, you can do almost anything. There is a tutorial video at the top of the documentation page. Being succinct is not my strong suit, so the video is pretty long. It has a lot of great information in it though.

https://varlab-dev.atlassian.net/browse/HY-133

Proposed Tutorial Steps

  1. Open the drawer - Have the user open the drawer.

  2. Select the Pressure Guage - Have the user click on the pressure gauge to place it on the board.

  3. Close the Drawer - Close the drawer by clicking on it.

  4. Move the Pressure Gauge - Click and drag the object around the board.

  5. Select the Pressure Gauge - Click on the Pressure Gauge to select it.

  6. Rotate the Pressure Gauge - While the object is selected, press the rotation buttons to rotate the object.

  7. View the Magnifier - While the pressure gauge is selected, view it in the magnifier. (notes A)

  8. Remove the Pressure Guage - Select the object and press the 'X' icon to put it back into the drawer.

  9. Place 2 Objects on the board - Open the drawer and place an FCV and FlowMeter on the board.

  10. Connect the Ports - Make a connection between the 2 objects. (notes B)

  11. Remove the Connection - Click and release on one of the ports to remove the connection.

  12. Connect a component to the HPU - Create connections to the Hydraulic power unit. (notes C)

  13. Move to the HPU - Click on the HPU to move to the camera to the HPU

  14. Turn on the HPU - Click on the switch to on the HUP. (notes D)

  15. Turn off the HPU - Click on the switch again to turn off the HPU

  16. Place a DCV and FCV - Open the drawer and select a Directional Control Valve and Flow Control Valve on the board.

  17. Select the DCV - Click on the DCV to select it. It is selected once there is a dark blue outline on it.

  18. Move the DCV Handle - While selected, click and drag the handle to move it back and forth.

  19. Select the FCV - Click on the FCV to select it. It is selected once there is a dark blue outline on it.

  20. Open the FCV Valve - While selected, click and drag on the handle to open or close it.

Feel free to alter these steps as you see fit, these are just an outline.

The steps should not be restrictive. I think we should lean more on the side of “A user accidentally completes a task” rather than “The task is too complex and the user can figure it out.” The tasks should be completed by the user doing something, not setting something to a set value. For example, the task shouldn't be “Open the FCV to 25%”, it should be “Move the handle on the FCV” where any movement completes the task. The ITaskables are not nessasarlly set up for this, but it should be pretty easy to change. See the PressureGaugeController class and how it tracks if it has moved to see how this can be done.

Tutorial Notes

A) This could be tracked with the Button Graph interface. More about the button graph interface can be found in the Graph Management documentation.

B) Tracking specific connections can be tricky. It can be set up that the task only passes when the connections on the board are set up in 1 specific way; see how the old Lab 1 & 2 graph scenarios set this up. We could also just have this task track when any connection is made. See how the PressureGaugeController tracks when it is moved around the board for an example of how this could be done.
The PortConnectionManager class is used to track the connections that are made. You should make it implement ITaskable to track the connections that are made.

C) You could apply the Auxiliary Highlight to the ports on the HPU so that they know where to make the connections to.

D) You can not make connections or remove connections when the HPU is turned on. We should make sure that this is explained to the user.

Using Interactable Subsets and Auxiliary Highlights

https://varlab-dev.atlassian.net/wiki/spaces/CE/pages/138805291/Interaction+System#Creating-a-Subset-of-Active-Objects
By using subsets of active objects, you can control what objects the user can interact with on the screen. This can be used with the Auxiliary Highlights to highlight the object the objects you want the user to focus on. There isn’t any official documentation for the Auxiliary Highlight, but the AuxiliaryHighlight class has some useful information in it.

Getting data about Components on the Board in the Graph Scenario

The Hydraulic Components that are placed on the board are instantiated at runtime. This means that we can not directly access them when we are building the Graph scenario. The ITaskable system will still work as expected, we just couldn’t call something like Interactable.SetSubsetActivated(tutorialObjects.PressureGaugeOnBoard); because there is no way that tutorialObjects could have a reference to the PressureGaugeController object that is on the board.
To get around this we could make a new method on the HydraulicsBoardController class. This method would return the HydraulicComponentController objects on the board. (note that Component and HydraulicComponentController are very different classes. The Component is used in the hydraulic circuit simulations, the HydraulicComponentController is the actual GameObject that is on the board. Make sure the classes you’re accessing are children of the right class for your purpose.) Add a lambda expression to the OnEnterVertex() for the task where you want to access the component on the board. Inside the expression, get the HydraulicComponentController from the HydraulicsBoardController. You can then directly access the object on the board while inside the expression. This could be useful for adding an Auxiliary Highlight or using it in the Interactable.SetSubsetActive().
