VARLab : Inspection element's visual representation in UI

Created by Rounuk Kumaran (Unlicensed) on Aug 10, 2022

There are three major ways of displaying information for elements that needs inspection throughout our simulation

  1. Images

  2. Gifs

  3. Videos

What is the difference in using these methods?

Each method that we utilize has its own pros and cons and is necessary to represent different types of inspection based on their strengths. They have their own benefit but also time associated with making it available for the devs to utilize.

The various pros and cons for the various methods shall be stated below.

1. Images

Use of images are the most basic thing we can use in showing good as-well as bad states for the inspection elements

example: (Will be Uploaded when it is ready)

Pros:

  • Images can be easily obtained especially for good-state of our inspection

  • Images can easily be manipulated to create bad-state by adding 2D decals

  • The time required to make these edits is less especially compared to video editing

  • The decals needed to add to the images are also readily available from massive sources online.

  • Images are relatively easy to load and will keep the sim lighter

  • Most of the inspections can be done with this method be it the good or the bad state

Cons:

  • Images, though informative, it doesn't give a very lively feeling. It makes for a less immersive experience in comparison to a gif or a video.

Images can be edited in free image editing software like Photopea which is a free alternative to Adobe Photoshop, which makes this method quite cost-effective

2. Gifs

Gifs are just an extra flare on an image. The whole idea is to utilize the base image and have elements that are actively inspected to be shown with more immersion. Gifs are basically a series of images that have alpha present in them thus helping overlay themselves on base images or videos.

example: (Will be Uploaded when it is ready)'

Pros:

  • Gifs are more immersive and can be utilized to show malfunctions that need active observation like smokes, electrical faults, leaks or gauge measurements, etc.

  • Gifs are just an advanced Image sequence that can include audio within its application

  • Gifs can be less intensive than a full video to be incorporated and still tells a story, it is a blend of using images or videos

  • Gifs can be manipulated using free online software like photopea, https://ezgif.com/ , https://www.flexclip.com/learn/add-gif-to-image.html , etc. which are easy to learn and implement.

Cons:

  • Gifs have fewer resources than Images that can be utilized well.

  • Gifs sometimes require quite some more work than images in editing the background out from what we require

  • Gifs can require more computing than images but surely less than videos

3. Videos

Videos are one of the best ways we can show immersion to the user. It is one of the best ways we can show a user the good state of an object and how it should actually look in a working condition.

example: (Will be Uploaded when it is ready)'

Pros:

  • Videos are good for showing most of the good states

  • Videos are the most immersive for a user and can actually contain a lot of information that can meet the learning objective of the user

  • The user has the ability to see the object in a more realistic manner where it can have more angles

  • Videos can have proper audio which makes for a splendid learning experience for the user

Cons:

  • Videos can only show most good states, but bad states need to be edited onto the videos

  • Though the bad states can be shown in the video too, the original object needs to be tracked in a Video compositing software like Adobe after effects which needs to be purchased and it requires multiple setups done to make things look believable

  • It is a very time-consuming process to make such video edits compared to image or gif edits

  • Videos might not be readily available and might need more space on the repo

  • Videos also might take longer to load for the user
