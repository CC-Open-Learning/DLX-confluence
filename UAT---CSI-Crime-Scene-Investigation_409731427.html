SharePoint Functional Test Results - P2.04 CSI - 28092021 WebGL

  • The original version of this DLS required download and installation. This was the version that was used for UAT in W21 term.

  • Overwhelmingly, the feedback that was received from the W21 UAT effort was that students and instructors were very happy with the content and experience, but very unhappy with the deployment method - they did not like downloading and installing everything.

Important UAT Feedback from W21 - Summary

Feedback

Response or Analysis

Decision

Students did not appreciate the timer. Many students indicated that they felt too focused on the time, felt rushed, did not like being thrown out of the simulation when the time ran out

Content Experts insist that the timer is adequate and appropriate. They are aware students don’t like it, but they feel it is a necessary part of the learning experience.

No action

Some students had trouble navigating, suggested adding to the tutorial

No very specific instances were mentioned, students that found it confusing still managed to complete the sim.

No action

Some complaints about mouse control and having a difficult time being precise with the mouse

This is mostly controlled by device configuration settings in the OS, but we could change the simulation to include device overrides for mouse sensitivity. It was reported as an annoyance, not a blocking issue

No action

Some complaints about unable to install/run the simulation due to device or OS limitations

Valid point - this sim requires minimum 4GB of RAM to run, recommend 8GB for improved performance

Update user documentation to specify minimum RAM requirements.

  • Given the pandemic situation forcing remote learning and our general lack of preparedness to support a download server or provide tech support at that level, VARLab decided the best bet was to ask XPertVR to rebuild the CSI sim using Jordan’s WebGL build process, and publish as a SCORM package.

  • Decision from VLT in August 2021 was that CSI does NOT require a repeat user acceptance test following conversion to WebGL SCORM deployment - we would do functional testing only. The learning content and user experience within the sim are not changing, so there is no need for additional user testing.
