VARLab : 2. Creating Maps.

Created by Colm Ambler (Unlicensed), last modified on Sept 10, 2021

Creating maps in RPG Maker is simple. The first step is to right click in the map index and choose new.

Once you do, you’ll be greeted with a host of options you can pick from.

Name: The name that will appear in your map index

Display Name: The name that will appear in the top left of the screen when the player enters the map.

Tileset: As mentioned, each tileset holds many tilesheets within and we can only pick one per map.

Scroll Type: This is used if you want the map to scroll endlessly either horizontally or vertically.

Width and Height: The width and height of the map, pretty self-explanatory.

Enc. Steps: Determines the amount of steps it’ll take to run into a random encounter. In the picture above, a random encounter will happen within 30 steps of each other.

Autoplay BGM and BGS: Determines if background sounds and music will play when the map loads. They carry over into other maps so don’t need to apply it to every one.

Specify Battleback: The backdrop image for battles.

Disable Dashing: Players can dash by holding shift when they move. That is, of course, unless you check that button.

Parallax Background: An image that will go behind every other layer and will act as the background for any space that has no tiles.

Note: These are usually used for plugins, and it depends on the specific plugin.

Encounters: Here, we determine what kinds of encounters players will find. For our purposes, these are left blank.

Tilesheets

A Tiles

B Tiles

C Tiles

A tiles are the trickiest because, as there’s a lot to them. It is made of up a maximum 5 tilesheets, labelled A1, A2 etc. A1 is for animated tiles, A2 are for ground tiles, A3 are for exterior buildings, A4 are walls and A5 are for more generic tiles. These are all labelled properly within the database so there’s no need to memorize them.

B Tiles are for any other sprites that don’t fall in line with A tiles. You have B tiles, C tiles, D tiles and E tiles for the same purpose.

As you can see, there isn’t much difference between B and C, but it’s good practice to make sure they relate to each other. For example, in the CCRPG any asset that signifies a transition to a new area is on a separate tilesheet.

An easier explanation could be that A are all the environment tiles, whereas B, C, D and E are everything else.

------------------------------------------------------------------------------------------------------------------------

Colliders

Inside the database is where you can set up colliders for each tile which can be below characters, on the same plane or above characters. You change them by clicking on the specific tile.

X refers to tiles that the player cannot walk on. They are on the same plane as the player. In the image, tiles such as the base of trees, windows and objects are marked with X

O is for tiles that the player can walk overtop, they are below the player. Open doorways and rafts are marked with O in this picture

The star represents tiles above the player. They can walk underneath them. In this picture, the tops of trees are marked with the star, giving the effect that the player can walk behind them.

I should also mention if it wasn’t obvious that this also determines how they will be rendered. Whether they’re above or below the characters is determined by their symbol.

------------------------------------------------------------------------------------------------------------------------

Layers

You’ll mostly likely spend most of your time in the automatic layers tab as is the case in the image above. It decides which layer tiles should be on determined by the placement (ie. if something is above the tile or below) and the collider value as mentioned above. This will also determine if the new tile will overwrite the previous one.

One thing to be careful of is with the colliders because tiles will still be behind others (specifically on tiles that aren’t on the A set.) Each tileset should have an empty tile for removing any tile data.
