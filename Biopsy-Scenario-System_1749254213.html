VARLab : Biopsy Scenario System

Created by User e88ce, last modified by User bf53c on Apr 15, 2025

RT’s Biopsy Procedure mode works by following a numbered list of defined sections. These sections are known as “Scenarios”. This document provides step-by-step instructions on how to create a scenario.

Creating the Scenario Folder:

  1. Determine where the scenario falls in the numbered order. The scenario should be represented by a scene in Figma so you can figure out the correct number by comparing the scene order to the scenario order.

  2. Create a folder in Assets/Scriptable Objects/Biopsy/Scenarios for the new scenario. It should be named “[scenario number].[scenario name] Scenario” (eg. “3.Initial Observation Scenario”). The [scenario name] usually isn’t defined, so name it something that appropriately summarizes what it represents.

3. Within the newly created folder, create another folder named “[scenario name] Scenario Tasks” (eg. “Initial Observation Scenario Tasks”).

Creating Scenario Tasks:

  1. Identify what “Tasks” are required for the scenario. Refer to the task document for information on the types of tasks and how to create them.

  2. Create all required tasks within the scenario tasks folder.

  3. Update the TaskIndexIncrement and ScenarioIndexIncrement fields of the created tasks:

Scenario With 1 Task: Generally, when a scenario has 1 task, the ScenarioIndexIncrement needs to be true for that task. This doesn’t need to happen when the task inherently updates the scenario index (currently only happens with the MessageTaskSO). The TaskIndexIncrement needs to be false.

Scenario With 2+ Tasks: When a scenario has more than 1 task, the TaskIndexIncrement needs to be true for all tasks except the final one. The ScenarioTaskIncrement needs to be false for every task.

ItemRequestTaskSO: These tasks use unique values (GoToNextTask and GoToNextScenario) instead of the general values. The previous rules are mostly the same, but if a scenario ends with an item request GoToNextScenario must be true as it does not auto-increment.

Creating The Scenario:

  1. Return to the scenario folder created earlier and create a ScenarioTasksSO by inputting Right-click > Create > RT Scriptable Objects > Biopsy > Scenario.

  2. Fill out the ScenarioName with the name decided on earlier. Also, fill out the ScenatioTaskList with the previously created tasks. Make sure they’re in the right order!

Referencing the Scenario:

  1. Find the BiopsyController in the project hierarchy.

  2. Update the ScenarioList to include the new scenario in the numbered order determined earlier.
