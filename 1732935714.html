Overview 

The Respiratory Therapy Orientation DLX was developed by CVRI (formerly VARLab) at Conestoga College from September 2024 until April 2025. The purpose of the DLX is to help students orient themselves to the location and purpose of all medications and supplies they will need to be familiar with when performing a bronchoscopy procedure. It is meant to be a 1:1 representation of the ideal stocking/locations for everything compared to the College’s real training procedure room.

The current release includes the Orientation module of the DLX which allows students to interact with and read/listen to descriptions of the items they will need to become familiar with. In the Orientation module, students can navigate the procedure room with the goal of finding and learning about 100% of the items. There is no time limit or optimal path. In future releases, we will include the Biopsy module which will simulate the actual procedure, setup, and post-procedure.

This DLX does not provide any assessment capabilities, nor return any grades to the LMS. Students can play an infinite number of times to aid their learning.

DLX Objectives 

The primary objective of the RT Orientation DLX is to provide students with unlimited access to the Procedure Room, the various medications and supplies that are used in class, and a majority of the equipment for the whole program. Students can use this in a number of ways when access to the real equipment is either heavily scheduled or unavailable (after hours).

Known Issues 

At the time of General Release (April 4, 2025) for Respiratory Therapy Orientation 1.0, the following low-impact issues were known to exist. For each issue, a workaround is provided, to assist users who may encounter these issues when using the DLX.

Issue # 

Description 

Workaround or Discussion

RT-486

There is a brief flicker noticed in the Object Viewer window when interacting with a bronch tower item.

Some work was done to improve the Object Viewer flicker experience. This issue is specific to interacting with bronch tower items, the scene is shown briefly in the OV window prior to the explorable loading into it.

RT-504

POI workflow can be disrupted when rapidly entering/exiting the menu

This is an edge case where if the user was to click on the menu button and exit the menu prior to the camera transition finishing a disruption of which POI’s should currently be active occurs. A fix has been suggested on this ticket.

RT-503

The order of items in the Progress Indicator List changes on reload.

Not a bug; however, to provide a more consistent experience, it may be better to have these items remain in the same order in the progress indicator list. We’ll want the order to match the other parts of the DLX as closely as possible.

RT-501

Shadows on OV items can hinder viewing.

When interacting with some items in the OV, depending on the orientation, the lighting/shadows can lead to parts of the object not being entirely visible.

RT-470

Optimize project structure for better loading.

The load times in the DLX are not as fast as they could be. There is a fair amount of clean up that could be done to decrease the file size and potentially aid the slow load times experienced in certain hardware and network speeds.

RT-300

Object Viewer does not rotate proportionally.

When interacting with items in the OV, the movement is applied a little more aggressively to the left. This results in the movement feeling slightly off for some users. During the student play testing, one student did mention they found it hard to reset the object back to its original position, as the movement can be slightly unpredictable. This is potentially something that needs to be resolved at the package level.

RT-510

The camera sensitivity slider can be dragged beyond the minimum possible value.

Dragging the camera sensitivity slider to its lowest value causes it to jump to a slighter higher value. This is because the value it jumps to is the true minimum value that the camera sensitivity can be. This causes no functional issues but should be changed in the future to avoid confusion.

UNIVERSAL

DLX cannot be played on mobile devices such as tablets or phones. 

Use program-recommended hardware to play this DLX. 

UNIVERSAL

Saving progress while playing offline is not possible. 

Assuming the DLX is loaded and begun while connected to the internet, the user may continue to play and explore offline. However, if the user does not re-establish network connectivity before closing the browser tab, then explorable progress will not be saved.

UNIVERSAL

DLX is not available without login to eConestoga. 

There is no workaround for this problem. User controls for eConestoga are managed by Central IT. Please open a ticket to gain access to eConestoga.
