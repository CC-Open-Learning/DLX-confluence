VARLab : Edge Case: Loading Image For Inspectable Door

Created by User aa77f, last modified on Apr 18, 2024

This article will be going over a specific edge case that is found on the fridge and freezer doors. The problem that was encountered was these doors are inspectable however when they are inspected they are open, and the default state of the doors is closed. This creates a problem when the game starts and wants to take photos of the objects that are in the save file the door is no longer in the same position.

The solution to this was to introduce a “second” door that is turned off and swaps with the original object on load.

To achieve this a stand alone component was created.

LOD0 (1) is a duplicate with all the same information in its inspectable component but does NOT have the swap component and is positioned in a way that matches the doors “Open” state, the Original object will have the swap component on it with the references to the original, and the object to swap to. On load if the DLX needs a picture of the door the objects are swapped, it gets the photo then swaps back. Please look at Door_Cooler within the Walk-In Fridge Exterior as an example if you need to implement this functionality.
