VARLab : Ticket Standard for HE Inspections

Created by Kristian Biviens (Unlicensed), last modified on Jan 10, 2023

Each normal inspection will consist of 4 related tickets for implementation. This will be consistent for 2D inspections and 3D inspections. It’s important that these tickets have the related art tickets attached to them as well as the other 3 tickets linked to them as well.

Importing Assets into Unity

The first ticket will be for importing assets into Unity. Any assets used by the inspection will be imported into the proper folders within the project structure. This is a good example of a ticket: DOZ-323 - As a developer, I need to add images, videos, and conditions for the oil disptick so the inspection can be set up Done

We can see it blocks the implementation if the inspection in ticket DOZ-211 - As a learner, I want to be able to inspect the oil dipstick so I can learn about the possible oil levels. Done . It also has the related art tickets linked as well. Once this ticket has been finished and merged in, we want to move the art tickets to DONE as well. This shows that they have been imported to Unity and no longer need to be reviewed.

Writing the Dialogue

The second type of ticket is for writing the dialogue for the inspection. The dialogue is written within this document in Sharepoint. This ticket will a developer ticket instead of a learner ticket. We won’t need a branch associated with it but it’s good practice to attach the written changes into the ticket itself. Another great example of this type of ticket is here: DOZ-279 - As a developer, I need to write the dialogue for the oil dipstick inspection Done . We see that there is an attached image of the script and it’s well commented. We also see that it also blocks the DOZ-211 - As a learner, I want to be able to inspect the oil dipstick so I can learn about the possible oil levels. Done ticket.

Generation and Importing of Voice Lines

A third type of ticket for the inspection is generating and importing voice lines for the simulation. The important part of this ticket is that the voice lines MUST be approved prior to import by our script expert. Having voiceover isn’t mandatory for the simulation so they will be given Low priority. A good example for this type of ticket is DOZ-280 - As a learner, I need to hear the voice lines from the approved script for the oil dipstick inspection so I understand what I have to do. Archived .

Implementing the Inspection

The last type of ticket is the implementation of the inspection. This will include adding the camera position for a 3D inspection:

Or adding a new box inspection for the 2D inspection:

Another good example of this type of ticket is: DOZ-211 - As a learner, I want to be able to inspect the oil dipstick so I can learn about the possible oil levels. Done . The containers created within the previous ticket is added into the inspection and the camera position or inspection box is created for the learner to actual inspect the item.
