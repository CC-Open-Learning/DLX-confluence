VARLab : Bodyswaps IT Support Requests Recap

Created by User cb87c, last modified on Nov 04, 2024

This report summarizes the IT Support Requests related to Bodyswaps received from October 1st to 4th, 2024. During this period, over 22 issues were submitted, marking a notable increase in support requests for Bodyswaps. Two key factors contributed to this increase:

  1. Bodyswaps is being utilized by more classes in the fall term.

  2. A system update (September 2024) from Bodyswaps led to some unexpected issues.

User logged out of Bodyswaps Application - Error “Unregistered Device”

Summary of Issue

When users opened the Bodyswaps app they were logged out of their personal Bodyswaps account. On the screen this error message appeared “Unregistered Device. This device is not authorized to run Bodyswaps. You will need to register this device.” When the user clicked “Continue”, the user was prompted to enter their access code to login on the app. This error confused a lot of users as they weren’t aware how to access their access code.

Number of Requests

10 Reported for IT assistance. Faculty reported that this happened with multiple students but they were able to resolve the issue on their own by finding their access code. Bodyswaps reported that multiple students reached out to them for support, waiting for them to send me the numbers.

Root Cause

Bodyswaps Support Team has confirmed that the 'Unregistered Device' error (Logged out of the app) is a result of the system updates that were performed on the backend - these updates included major architectural updates as well as updates to their servers. As a result of the updates, learners will need to re-identify themselves within the app. Once re-identified, the error should not appear again. - This was the update that was done of September 11th I believe, I know that Lisa Trimble sent a message about it in one of the teams chats but it didn't seem like a concern as students are directed to use the most up to date app according to Corey. In the email it didn't mention to re-enter your access code, but it seems that we had too even if we had the updated version of the app.  

Resolution

Users had to re-enter their access code and then would have full access to the Bodyswaps app.

A lot of users didn’t know how to find their access codes, IT OPS/leadership/faculty helped provide access codes and directed them on how to find the access code in the future.

Class Enrollment Link (Class Invite) Missing Class ID

Summary of Issue

Faculty and students reported malfunctioning Class enrollment links where the Class ID was missing, it was an empty field. Users tried to manually enter the Class ID in the empty field but you can’t type in the field. Class enrollment links are for students to self enroll into a class but they couldn’t self enroll with the Class ID missing.

Number of Requests

4

Root Cause

Back in September Bodyswaps performed a system update, where they made some changes behind the scenes. Any Class Enrollment Links that were created prior to the system updates were regenerated during the update. Which caused the class ID to be missing in some cases and causing self enrollment issues for faculty and students.

Resolution

Class Enrollment Links need to be updated on any documentation that is saved on eConestoga or that was sent out to the students.  

  • To obtain your new Class Enrollment Link:

  • Navigate to the Classes tab on the Bodyswaps website

  • Navigate to your class in the dashboard

  • Select the 3 lines icon on the right hand side of the page > Select class enrollment link to view the link

  • Right click on the link > Copy link address

  • Now use this new link to replace the old link in any documentation that was shared with the students

Students trying to login to Bodyswaps using the Class Invitation Link(One time use for self enrollment) Error Code 401

Summary of Issue

Users are trying to login to the Bodyswaps website with the link that was used for self enrollment. The self enrollment link is a one time use link, if a user tries to use it a second time they won’t be able to login, error code 401 appears.

Number of Requests

5

Root Cause

User error. Users trying to login to Bodyswaps with the same link that was used for self enrolling into a class.

Resolution

Students should go straight to the Bodyswaps app that was previously downloaded to use the modules. Their account should be left logged in, if they ever need to re-enter their access code that is when they would access the Bodyswaps website: https://go.bodyswaps.co/

If students are trying to access the Bodyswaps website they should use the normal URL not the self enrollment link.

It gives an error on the users email address because they have already signed up using the class link.  The link above is the Bodyswaps website and they should be able to login with their email and password.

Redirecting Error - Can't access dashboard and modules or anything else

Summary of Issue

Users experiencing a “Redirecting…” error when trying to load any page on the Bodyswaps website. Users are unable to access the dashboard, modules or anything other page. Users tried to refresh the page and log out and log back in but they still receive the same error.

Number of Requests

3

Root Cause

Unknown. Potentially related to the updates Bodyswaps performed. As this issue didn’t happen before even if third party cookies were blocked.

Resolution

Clearing cookies and cache, and allowing Third Party Cookies resolves the issue. Once these steps are completed users are able to access all of the Bodyswaps website without the “redirecting…” error.

Steps to take:

  • Navigate to History > Delete Browsing Data >Basic

  • Select “Cookies and other site data” and “Cached images and files” and then select Delete Data

  • Navigate to browser Settings > Go to Security & Privacy > Go to Third-party cookies and select Allow Third Party Cookies. Alternatively you can add Bodyswaps to Sites allowed to use third-party cookies.

Help article Bodyswaps created:

https://support.bodyswaps.co/knowledge/bodyswaps-go-keeps-redirecting-me

Conclusion | Preventive Measures

  • Update training documentation: Provide additional information in Bodyswaps training documentation for faculty and students.

    • How to access your individual access code (Student and Faculty)

    • How to generate a students access code (Faculty)

    • How to generate a class code (Faculty)

    • How to access Bodyswaps after you have registered. (Class Enrollment Link is one time use)

  • Increased Monitoring: Regular monitoring of system performance, especially after system updates to identify and address issues proactively.
