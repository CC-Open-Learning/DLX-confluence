VARLab : Game Analytics Sandbox Experiment

Created by Germán Gaussmann, last modified by David Inglis (Unlicensed) on Sept 22, 2022

For this experiment we’ve set up a sandbox project hosted in Bitbucket (https://bitbucket.org/VARLab/campus-rpg-sandbox/src/main/) by creating a new map in Campus RPG and removing everything but the things needed for the experiment:

  • The Condor, Kaedama, and Businessman NPC assets

  • Quest log plugin

    • To be used as a reference on how the devs use plugins to trigger events

  • Grassy plains map

Objective

A successful deliverable for the experiment will:

  • Create an interface between Game Analytics' SDK and RPG Maker

    • That allows the Campus RPG devs to send customizable signals to Game Analytics

    • It provides Game Analytics with an accurate count of events being triggered

  • Show on Game Analytics that we are able to detect the signals being sent

  • Confirm if Game Analytics can provide us automated information like length of the session (time) or if that is something that we will have to calculate on our own

Nice to have

If we have enough time, these are some extra points we might want to explore

  • Does Game Analytics provide us with extra information from our playerbase

    • General geolocation (we don’t need extreme accuracy, if we can get which country they are playing from would be enough, to see if we are getting potential international students to play before coming to Canada)

    • Desktop web vs. web Mobile
