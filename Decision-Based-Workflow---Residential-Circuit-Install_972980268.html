VARLab : Decision-Based Workflow - Residential Circuit Install

Created by Stacey Dineen (Unlicensed), last modified on Mar 15, 2024

The goal of this page is to track the decision-based workflow for TE Residential Circuit Install.

Target Playtime for 1 complete scenario: 45-60 minutes max (includes any tutorial time and results)

Primary Objective: Practice Students will have a lecture to introduce and discuss the project, and then they will have 6 days to practice that project in the virtual practice environment of this DLX. The primary goal of the DLX is to help prepare students for success in their in-person skills lab, which takes place one week after the lecture. Students may access the DLX at any time and replay is unlimited, but we are expecting highest usage and peak interest in playing the DLX to occur during the one week timeframe between the lecture and the skills lab.

Workflow Format: Task-based Multichoice

This DLX is a flatscreen procedural workflow for students to practice making decisions essential to the success of a residential bathroom electrical circuit. Students must select exactly one option from each set of options that are offered so that they can progress through the full workflow.

Students may not select any action that is not offered as one of the options.

Students may not combine options by selecting multiple options.

Task order is not pre-determined, the student will be allowed to select tasks in any order. The only exceptions may be where real physical dependencies exist (e.g. it is not possible to install a lighting fixture without a light box). In those cases, we may wish to limit the available tasks to improve the user experience.

Inspections must be triggered by the student - they do not happen automatically. If the student calls for an inspection when the work is not ready to be inspected, they will not receive an inspection report. It is up to the student to decide when each inspection should take place, but there is no limit on the number of inspections the student can call for.

Task Hierarchy - “Rewind” Feature for Fixing Mistakes

It is essential that the student is allowed to make mistakes and fix those mistakes as a way of building confidence and gaining experience in the residential circuit install workflow. For that reason, all tasks in this workflow are hierarchical, so that every task is aware of its dependent tasks. At any point in the workflow, if the student goes back to an earlier task and makes a change, the workflow will automatically “rewind” any dependent tasks and require them to be redone as well. This is not a chronological “rewind” - it does not matter what order the tasks were completed. When a task is undone by the student, only the dependent tasks will also be undone. Unrelated or non-dependent tasks will remain unchanged.

Success Criteria:

  • Electrical code rules (CEC and/or OESC - These are not in-game resources, students must reference their own copy of the code rules outside of the DLX. The student receives code rule compliance feedback in the form of Electrical Inspection results, where any violations are specified by code rule number. There are two inspections, rough-in and final, and both must pass for the student to complete the project. Students must remediate any code violations and repeat the failed inspection as many times as necessary until they pass. Example: CEC rule ### violation would be listed in the rough-in inspection report if a student installed the gang box too close to the shower. The student would need to fix this mistake and repeat the rough-in inspection to proceed with the project.

  • Project plan - This consists of a project plan (layout, details about required devices or fixtures) and a schematic wiring diagram. These are available in-game for reference at any time during playthrough. The student receives this feedback in the form of “homeowner” or “boss” feedback, which may be provided at irregular or unsolicited times in the workflow. This feedback is NOT related to CEC or OESC code rules, but is instead focused on conformance to the project plan and industry best practices. Example: The “homeowner” would be angry if the electrician installed their dining room chandelier in the bathroom. This is not a code violation, but it is still a relevant mistake that needs to be fixed. The “boss” would be upset if the student used 3X the number of staples as required to secure the cable to the studs, but this would not be a code rule violation.

This sample workflow illustrates a few key points:

  1. The number of options may vary for each decision - we need to be prepared to display a smaller (2-3) number of options as well as a larger (6-8) number of options, depending on the task.

  2. Every option has pre-determined feedback that is immediately written to the feedback manager. Only CORRECT options do not output any feedback.

  3. A selection in one task (e.g. box type) can determine which future task will follow (e.g. Select Fan Box Location). This will allow us to offer meaningful, realistic choices for each subtask based on previous selections.

  4. There can be more than one CORRECT answer. Note that in this example, it is acceptable to install the gang box to the right or the left of the entry door.

  5. A single option may result in more than one feedback result. Note that Select Gang Box Location in this example offers option #2, which results in both CODE and PROJ feedback items.
