Inspection Review UI Window

Test

Expected Outcome

Result

Click the 3 “Tabs” Non-Compliance Log, Activity Log, and Gallery.

the clicked on tab should switch styling to yellow with a thicker bottom border to visually show the user which tab is selected

Clicking the toggle(check box) should enable/disable the submit button.

The button should be enabled/disabled based on the state of the toggle.

Open the window, click the toggle(check box) close the window, then re open the window.

The toggle should be unchecked when the window re-opens to ensure the user must confirm their submission.

Open window, click top right X, Open window again, click bottom right “Close” button.

Both buttons should close the window.

Inspection Review-Gallery UI Window

Test

Expected Outcome

Result

A Integration/System level manual test with the code coverage report running, this involves taking 5 photos, then opening the Inspection Review window, navigating to gallery and clicking edit, cancel, highlight and un-highlighting images, clicking delete and testing the cancel and confirm options.

Behaviour should work as intended, with rows of images populating based upon location, when edit is clicked and images are clicked they should highlight and have a check mark, when clicked again it should remove those, when Delete is click a confirmation window should appear allowing the user to confirm or cancel, cancel takes the user back, otherwise the user clicks confirm and the images will be deleted.

Of importance is the GalleryBuilder file being covered completely by the test.
