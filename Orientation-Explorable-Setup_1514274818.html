Types of Explorables

There are two types of explorables in the Orientation segment. First are the ones that are always in the scene, and must be explored by the learner navigating to them in the scene. Currently, the only ones like these are the Bronch Tower explorables.

The second type of explorable and the more abundant one are the ones in the inventory. These are not active on the scene (they’re in the scene, just not active) and only become active while the learner is viewing them in the Object Viewer panel. They can be found on the Hierarchy under the “InventoryObjects” GameObject.

“InventoryObjects” is the parent object that holds everything. Below that we have 3 more game objects, each for holding a category of inventory objects. These are just for organization. They serve no other purpose. Lastly under these GameObjects is where the actual explorables are.

They are inactive by default as we do not want every single one to be active, it would cause a lot of lag, so they’re all set up and just left inactive.

Explorable Setup

The process for adding the explorables is very similar, no matter which way you are doing it, so follow this tutorial for both, and any differences will be noted.

Find the explorable model in the project’s game assets and drop it onto the scene. Right click the object, under Prefab, select Unpack Completely.

Components

Both types of explorables will need a common set of components in order to function properly:

  • Explorable (so it can be explored)

  • Interactable (required by Explorable, comes with it)

  • World Object (for Object Viewer)

  • Rotate Object (for Object Viewer)

  • Box Collider (Required for Object Viewer scripts)

Here are two images showing the setup for both scene explorables (left) and inventory explorables (right). As can be seen, the only difference is that the scene explorable has an “Outlineable” component, but this will be mentioned in more detail later; you can leave it out for now if preferred.

Component References

Now that you have all the components on your GameObject, you need to pass the scripts all the necessary references to function properly. Luckily, only one script needs references, and it is the same no matter what type of Explorable it is.

The script that needs some more information is Explorable. When looking at it, you’ll see that it needs 2 different scriptable objects.

The 2 Scriptable objects that it needs are the following:

  • ExplorableInformation - This one holds all the basic information about the explorable, its name, description, sprite (used for inventory) and the category it belongs to.

    • Navigate to Assets/Scriptable Objects/Explorables and create a new Scriptable Object (Explorable Information SO) name it after the item you are adding.

    • This SO needs the following.

      • Sprite - checkout image_capture branch, open “Photo“ scene

        • Drag model into scene, position in view of camera, take screenshot

        • The image needs to have 145x145 aspect ratio

        • Add the captured PNG file to Assets/Art/Images/Inventory Item Thumbnails

          • In the Inspector window, change “Texture Type“ to “Sprite (2D and UI)” and change “Sprite Mode“ to “Single”

          • You should now have a generated sprite to use for the “Explorable Information SO”

      • Category - Set from dropdown

      • Task Name and Description - This one holds the task name and description

        • This information can be found here - RT Script

      • Explorable Audio - The audio clip that plays for the explorable

        • Use these websites for the process of making an audio clip

          • Guide for making voice clips

          • Respiratory Therapy Script

          • Upload the new clip to - VoiceClip SharePoint folder - and link in the above RT Script xls

          • Add the clip to the appropriate folder under Assets/Audio - add it to slot in the “Explorable Information SO” you have created

  • Transform Details - This is a custom set of rotation and scale values that the explorable gets set to when entering the Object Viewer panel

    • Navigate to Assets/Scriptable Objects/Explorable Transforms and create a new Scriptable Object (Explorable Transform Details) name it after the item you are adding + “…Transform Details“

    • You may have to edit the scale and rotation values based on how your object appears in the game

Scene References

At this point, you should have the Explorable itself all setup. The last step is putting it in the scene and finishing any other script references that are needed for the Explorable. Both types of explorables have different things that need to be done in order to function properly. Please go to whichever explorable type you have (scene/inventory) and finish the last few steps.

Scene Explorables

The two differences that need to be done for scene explorables and NOT for Inventory explorables are the following

  • It NEEDS to have an “Outlineable” component, so it can be highlighted.

  • The explorable has to be added to the list of Interactables, which can be found under EventSystem and on the “Interaction Handler” script component

These two extra steps allow the explorable to be outlineable when hovering over it in-game.

Inventory Explorables

Inventory Explorables have a different set of tasks that they need done to function properly. These changes should NOT be done to scene explorables, only Inventory Explorables.

  • The GameObject needs to be added to the correct list under Managers > UIManager > Inventory. On the Inventory script you will see 3 lists, one for each category. Add the GameObject to the correct category it falls under

  • They need to be added to the “InventoryObjects” GameObject in the scene, under the correct category, and the GameObject needs to be set to inactive, this can be done in the inspector, beside the GameObject name, uncheck the checkbox.

  • Once this is done, drag the new object from the hierarchy into the prefabs folder under the appropriate subdirectory.

Finishing up

Lastly, make sure you update the GUID Manager so that the explorable will work correctly with the Save & Load system. To do this, go to the scene hierarchy and navigate to Cloud Save System > Guid Manager script. On this script, you should see a big button that says “Update Tracked Objects.” Click it. This will update the list of items being tracked for Save & Load.

You should now have a functioning explorable in the game. Make sure you check it is functioning how you want it to, and no errors are being thrown. The following are some things you should ensure are working:

For Scene Explorables

  • Does the explorable have an outline when hovering?

  • Does the Object Viewer panel open when clicking on the object

  • Is all the information on the panel correct?

  • Does the audio clip play when clicked on? Is it the correct audio clip?

  • Does this new explorable worth with Save & Load?

For Inventory Explorables

  • Does the Explorables image show in the Inventory?

  • Does the inventory correctly update when exploring the explorable?

  • Does the Object Viewer panel open when clicking on the object

  • Is all the information on the panel correct?

  • Does the audio clip play when clicked on? Is it the correct audio clip?

  • Does this new explorable worth with Save & Load?
