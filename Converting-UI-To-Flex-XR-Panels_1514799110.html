Introduction

This document will go over the steps taken to convert the User Interface into flex XR panels. Keep in mind that this document is a result of research and not to be considered the optimal solution. Rather this is the solution that fit the requirements and time constraints for the time.

You should have already had the project set up for VR before this. including adding the CORE XR package.

Steps To Convert UI

  1. Navigate to “Packages → Core XR Tools → Runtime → FlexXR → Prefabs”

  2. Locate the FlexXR Panel prefab and Place this object in your scene

  3. Navigate to “Assets → Prefabs → XR” if this folder doesn’t exist create it

  4. Drag the FlexXR Panel you placed in step 2 into this folder to create a prefab variant. This is what you will be working with when ever you need a panel.

  5. Navigate to “Packages → Core XR Tools → Runtime → FlexXR → Resources”

  6. Copy the “FlexXR World Material”

  7. Navigate to “Assets → Art → Material → XR” if this folder doesn’t exist create it and place the material here

  8. Select your copied material and set the shader. search “Unlit” and set it to the universal rend pipe line version

  9. Set Alpha Clipping to true and the value to 0.5 (1 makes the ui transparent and 0 makes the blank spaces in the UI a opaque Black wall)

  10. Navigate back to your FlexXR Panel variant that you made in step 4 and begin editing it.

  11. Set World material to the material you created in step 7.

  12. In the Runtime section set the Interaction mode to Mix Reality.

  13. Fitting mode should be set to “Crop to Flex XR Content” but was receiving errors about vertices. this SHOULD BE “Crop to Flex XR Content” but just to avoid this error i have temporarily set it to Fill Render Texture.

  14. Now the Flex XR Panel Variant is set up you can use it to convert your UI’s.

  15. Place the new variant in the scene.

  16. Drag your UI’s UI Document component into the “FlexXR Panel Variant→ Flex XR Panel Manager(script) → Document” to link this panel to that document you are converting.

  17. navigate to the UI Documents Source Asset of the object you’re converting and Open it for editing

  18. Under Library → Project → Custom Controls → FlexXR → Runtime → FlexXRPanel. drag the FlexXRContainer into your Hierarchy

  19. Place this container as the root of the document so that everything else is a child of this container. Like so:

  20. Save and close this document. At this point your Panel should be functional.

  21. If you are reciving “Assertion Errors” follow the step(s) below:

  22. Under the original UI’s objects UI Document, set the Panel settings to “FlexXR UI Panel Settings”.

Known Issues

Flex panel gets in the way of other panels when nothing is there:

Problem: the panel even if it is not displaying UI elements is still there.

Fix: have some sort of function/call that will disable the lowest level of the XR Panel the “Flex Mesh” when the display style of the ui = none

Flex panels gets in the way of other panels when its not even overlapping

Problem: This is due to the panel being much larger than what it requires. usually due to some sort of sizing error

Fix: Ensure your flex XR Container is sized properly to only contain the UI Elements it needs to in the UXML document. Also set the Fitting Mode of the Flex XR Panel Manager to “Crop to Flex XR Content” as explained in step 13

Can’t interact with the menu

Problem: many possibilities. could be overlapped with UI when it pop’s up making your controller not realize it’s overlapped (just point away then point back at the ui). another panel could be blocking your interactions. More likely you have the Interaction mode set incorrectly

Fix: First make sure you have followed step 12. the different interactions modes behave differently. Flat screen is with mouse on a flat screen. World is your mouse interacts with the object in the world. Mixed Reality is your XR controllers interact with the panel using your triggers.

if this doesn’t fix it check that you don’t have another panel infront of this panel as it would hit your raycast first blocking interactions

Panel Isn’t displaying when expected

Problem: the panel display settings aren’t changing when they are expected to. this is likely caused by another line of code being blocked when converting to UI references to things such as Main camera, point and click navigation and player controller are being held up by a line of code that no longer runs properly.

Fix: follow the code execution and determine whats is happening during each line. check for a line of code that might be blocking execution or failing due to a reference no longer existing. often during this research it was a line related to player controller, point and click navigation, or Main camera.

Also double check that it isn't in your unity events as these will still cause issues but won’t throw an error

UI on panel looks squished or non-functional

Problem: No clue. not enough time to research properly. probably a sizing issue with the Flex XR Container.

Fix: try to resize the UXML document to properly encapsulate the other visual elements.
