VARLab : Network Test Procedures - MPC

Created by David Inglis, last modified on May 08, 2024

Overview

These baseline tests intend to document network and web browser performance of the Motive Power Circuits DLX as a WebGL module in eConestoga.

These test procedures are being performed on the MPC-24W8.5-3MAY2024 series of builds which are proposed as release candidates. All 9 independent lab modules and the Digital Twin module are close in file size, with the smallest being the Digital Twin at 20.9MB, and the largest being Circuits Lab 5 at 35.8MB (compressed).

Server Load

The web host for all modules is eConestoga so no measurement can be performed server-side. Loading the simulation itself is dependent on eConestoga servers and the Unity WebGL runtime.

Latency

Analytics events are “fire-and-forget” and the simulation is not dependent on a response from these API calls.

Average request time: 247ms

Packet Loss

Analytics events sent while the simulation is offline are lost.

Single-Player Experience Tests

Single-Player Experience Tests intend to validate the stability and support of single-player experiences with the sim.

Load Times

As noted in response times above, with an ~average~? network connection one can expect to load and launch the sim within the magnitude of one minute.

Fullscreen

All modules are intended to be launched using the “Open In New Tab” option for SCORM/xAPI content. The modules can be launched as embedded content within the eConestoga tab, and the screen will resize to fill the available space.

Whether in a separate tab or embedded in the page, learners can resize the browser tab and the simulation will resize its playable window accordingly. They can also enter and leave fullscreen mode with no issues.

Saving Progress

Saving progress is not a requirement for the Motive Power Circuits simulation. As there is no save & load capability, learners who close their web browser mid-lab will have to start from the beginning of the lab.

Offline

Loss of network connection does not interrupt the learner’s ability to play the sim. However, analytics events sent while the simulation is offline are lost.

Bandwidth

Lower bandwidth increases overall load times but has no effect on the simulation once loaded, as there are no dependencies on external network resources after the initial load.

Multiplayer Experience Tests

Motive Power Circuits has no multiplayer component, so no tests of this kind are being performed.
