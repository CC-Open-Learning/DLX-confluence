Introduction

The following documentation outlines the steps required to properly set up a Poi within the simulation environment using a reference example. It covers the necessary configurations for the Poi and can be used as a template for adding more Poi in the future. It also addresses potential issues that may arise from the current implementation and that includes any workarounds and suggestion that can be used to mitigate them.

Poi Setup

For this documentation, we are going to use the Piercing Tray Poi that already exists in the project as an example.

Then create a prefab of the Piercing Tray Poi with all the necessary child objects and placing it as a child of the Inspectables and POIs object as follows:

Ensure the Poi object prefab is centered in the Poi room/area so child objects will follow suit with the pivot setup for the Poi object.

Any changes to child object transformations will be an offset from the pivot center of the Poi object prefab.

Ensure the Layer for this prefab is set to Ignore Raycast.

In the screenshot below, Inspectables object has Poi specific interactable items in the simulation; Any new inspectables will be added here. The Navigation Waypoints object will contain all of the Point and Click waypoints within the Poi (You would need at least one waypoint within the Poi for the setup). The Environment object will contain the props within the Poi.

Some of the Poi may have a LevelGeo game object and this contains Level Geometry objects and other interactable objects like doors.

Box collider

You will also need to attach a box collider component to the prefab to define an area where if the player is within, will be able to do Poi specific actions. Set up the box collider component as shown below.

The box collider should only take up enough space at the bottom of the room that is deemed part of the Poi. This can be clearly seen in the above image that the Piercing Tray Poi is ONLY that specific area.

Once complete, you now have a Poi that has all of the required components for the simulation. Adding inspectables to the Poi will be covered in another documentation.

Issues and Risks

Multiple Poi in the same area

  • It could be a little unintuitive to have box colliders for Poi that are within a few steps distance from each other especially in the same room. This was noticeable in the Piercing Service Area Poi where I could not inspect an object that is right next to me because I was not in its Poi.

  • A potential fix for this would be to merge Poi that are next to each other and somehow retain the uniqueness of the Poi visits and functionality as is.

Inspectables within Poi box collider

  • Adding an inspectable within the box collider of the Poi interferes with the inspection workflow even though the box collider for the Poi is set to ignore raycast.

  • A workaround used was to set the box collider height of the object to be higher than the Poi box collider so that the raycast can hit the target object as shown below.
