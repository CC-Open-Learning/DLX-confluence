VARLab : Bulldozer - Making Changes To The Heavy Equipment Inspection Package

Created by Armando Interaminense on Aug 14, 2023

This page instructs how to make changes in the Heavy Equipment packages while inside the Bulldozer project. In order to modify and make permanent changes in the Core Inspection package you need to follow a few steps. This document will show how to do it.

 Instructions

1

Clone the Core Inspection repository to your Sourcetree

2

Create a local branch with the exact same name as the branch you are working on in Bulldozer.

Example: The name of the branch I was working in Excavator was called EXC-544-can-not-make-a-selection.

After creating a branch in Sourcetree with the same name as the branch you are working on in Heavy Equipment DLX, push it to remote.

3

Go to your Heavy Equipment DLX package folder, and open Manifest.json. Then, update Heavy Equipment Package by adding the name of your branch preceded with 'upm'.

4

You can confirm you are using the correct package in the package manager window.

5

After you make all your changes, commit, and pushed them, it is time to bump up the version. To do that open the package.json in the Assets/Package folder and bump up the version. In this example, we would bump it up to version 1.4.1.

6

In Bitbucket, go to Branches and open upm. Then open the last commit. And finally, create a tag with the exact same new version.

7

Finally, don’t forget to update the Heavy Equipment DLX packages as soon as your branch for Core Inspections is merged.
