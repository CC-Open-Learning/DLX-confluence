VARLab : Importing Art Assets

Created by User 0a15c on Apr 24, 2024

Introduction

This article covers the process used in PHI to import art assets into Unity. It follows a similar process covered in the Asset Import Lunch n' Learn<linkable?>, with the use of a tool developed to streamline the process. This article and process is up to date as of 16/04/2024. Numbered steps describe the default workflow, and lettered sub-steps describe potential variants of the default workflow.

Before you Begin

This article assumes you have a basic working knowledge of navigating the Unity Editor, particularly the Inspector, Hierarchy, and Project tabs.

 Instructions

Downloading the correct Files

  1. Locate the Jira ticket for the asset you are importing. It will be linked to the associated asset import ticket. If it is not correctly linked, it will be located on the PHI Art Active Sprint board in the Import column.

    In the Jira ticket for the asset, locate the SharePoint link to the files in the comments.

  1. In SharePoint, navigate to Engine Imports->Models. Download the .fbx file located there. Return to Engine Imports, then navigate to the Textures (Compressed) folder. Download the _colour.png, _normal.png, and _MRO.png.

    1. In the case where there are multiple .fbx files in the Models folder, download them all. This is a multiple-part model that will need to be assembled in editor.

    2. In the case where there are multiple Textures (Compressed) folders, download the colour, normal, and MRO file for each. This is an asset with multiple texture variations for the same model.

Importing into Unity

  1. In the Unity Editor, open a Project tab, and navigate to Assets->Art->Textures. Create a new folder named after the asset you are importing, then drag the downloaded colour, normal, and MRO files in it.

    1. In the case of a model with texture variations, instead create a folder named after the model, then a sub-folder for each texture variation. In each of these variant folders, place the colour, normal, and MRO for that variant.

  1. In the Project tab, navigate to Assets->Art->Models. Drag the downloaded .fbx file here

    1. In the case of multiple .fbx files, create a folder first named after the asset, then place all the .fbx files there.

Creating the material

  1. In the folder you placed the textures, select the color, MRO, and normal files you just imported. In the menu bar at the top of the editor, select VAR Tools->Create Complex Material. This will create the material that can be applied to the model. To verify it has been created, search [asset_name]_Mat in the Project tab.

    1. In the case of a model with texture variations, follow the above process for every variant.

    2. In the case of a model that requires a solid and transparent material, you need to create a transparent variant. To do this, right-click the created _Mat file and select Create->Material Variant. Rename the created variant [asset_name]_Alpha. In the inspector, locate Surface Options->Surface Type, and change it from Opaque to Transparent.

Adding Materials

  1. If you are importing an asset with no variations, follow step a. If you are importing an asset with multiple variations, skip this section and follow Creating the prefab 1-1.a.

    1. No Variations
      In the Project tab, select the .fbx file. In the Inspector, locate Materials->Remapped Materials. In the box with None (Material), drag the created material and select Apply. If there is an Alpha and Mat material for the asset, drag the Mat material to the box labeled _mat and the Alpha material to the box labeled _alpha. If the asset has multiple .fbx files, follow the above process for each.

Creating the prefab

  1. Drag the .fbx file from the Project to the root of the Hierarchy. Right click it, and select Prefab->Unpack. If necessary, rename the asset in the hierarchy to match the style of other assets in the project.

    1. If the asset has multiple texture variations, duplicate the unpacked asset in the hierarchy for each variation. In the Inspector, locate each element of the asset that has elements in Mesh Renderer->Materials. Drag matching Alpha and Mat materials to each one. Rename the root object to match the variant created, and repeat the process with each variant.

Ensure you check every element and sub-element of an asset for Mesh Renderers, and that materials are added to all of them. Take care to apply materials to all LODs.

b. If the asset has multiple .fbx files, drag all of them into the scene, unpack them as in step 1, and fully expand any with sub-elements. Using object names as a guide, arrange components to be children of matching loc_Attach elements. After each placement, in the Inspector, zero out the object’s transform.

  1. Select the root of the asset in the hierarchy, and ensure its Transform in the Inspector is zeroed out. Drag the asset from Hierarchy to Project, placing it in Assets->Prefabs. Delete the asset from the scene.

    1. For an asset with multiple texture variations, repeat this step for each variant.

Validation

Drag the asset from Project and place it somewhere in the Scene.

  1. Look at the object from multiple angles. Do the textures look correct? Pay close attention on models that had both Alpha and Mat materials, verifying that to correct components are opaque and translucent.

  2. Look at the object from multiple distances. If the object has different LODs, ensure that textures correctly appear for each LOD.
