Introduction

This guide explains how to set up the DoorTrigger prefab for an existing door, so that it automatically opens and closes when the player approaches.

Key Prefabs:

  • Door Object (e.g., DoorOrigin): The existing object representing the actual door model that will rotate. This should already be placed in the scene.

  • DoorTrigger.prefab: A pre-configured prefab containing the door trigger logic (DoorTriggerVolume.cs), the trigger area (Box Collider), etc. This is what will be added.

Setup Steps

Step 1: Add DoorTrigger Prefab

  1. Identify the Target Door: In the Hierarchy window, navigate to the existing GameObject that represents the physical door you want to make interactive (e.g., DoorOrigin).

  2. Place the DoorTrigger Prefab:

  • From the Assets/Prefabs/ folder, drag the DoorTrigger.prefab file into the Hierarchy window.

  • Make the DoorTrigger prefab a sibling to the target door object (i.e., place it under the same parent GameObject as the door object identified above).

For room doors within POIs, both the door object and the trigger is usually placed under a LevelGeo GameObject.

Step 2: Adjust the DoorTrigger Collider Area

  1. Select the DoorTrigger GameObject you just placed in the Hierarchy.

  2. Find the Box Collider component in the Inspector window.

  3. Adjust the Size and Center values of the Box Collider component to define an appropriate range in front of and behind the door.

Step 3: Configure the DoorTriggerVolume component

  1. Keep the DoorTrigger GameObject selected.

  2. Find the Door Trigger Volume (Script) component in the Inspector window.

  3. You need to configure the following fields:

  • Door: (Required Connection) Drag the door object from the Hierarchy onto this field. The door will not move without this connection.

  • End Rotation: (Required Adjustment) Enter the rotation angle value for when the door is fully open. This value differs for each door.

  • Start Rotation: (Required Adjustment) Enter the rotation angle value for when the door is fully closed. Like 'End Rotation', you must verify and set this value correctly for each specific door.

  • Speed: Controls how fast the door opens and closes. The default value is 5, which is usually sufficient. Adjust if needed.

Troubleshooting

Door Doesn't Move:

  • Double-check that the Door field in the DoorTriggerVolume script is correctly linked to the actual door object's Transform. If it's empty or linked incorrectly, the door won't move.

  • Verify the Box Collider area isn't too small or misplaced relative to the player's path.

  • Ensure Start Rotation and End Rotation values are different.

Door Rotates Incorrectly:

  • Double-check the Start Rotation and End Rotation values.
